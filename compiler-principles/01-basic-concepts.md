# 01. 基础概念 (Basic Concepts)

### Assembly Language / 汇编语言
- **中文**：汇编语言
- **定义**：用助记符表示机器指令的低级编程语言，与机器语言一一对应
- **说明**：比机器语言易读写；需通过汇编器转换为机器码；与特定 CPU 架构相关（x86、ARM、MIPS 等）
- **相关**：Machine Code（机器码）、Assembler（汇编器）、Instruction Set（指令集）

### Back End / 后端
- **中文**：后端
- **定义**：编译器中与目标机器相关的部分，负责代码优化和代码生成
- **说明**：接收中间表示（IR），输出目标代码；包括：机器无关优化、指令选择、寄存器分配、指令调度
- **相关**：Front End（前端）、Middle End（中端）、Code Generation（代码生成）

### Bootstrap / 自举
- **中文**：自举
- **定义**：用某种语言编写该语言自身的编译器的过程
- **说明**：通常先用其他语言（如 C）实现编译器，再用该语言重写自身；证明语言可用性；经典例子：Go 语言编译器
- **同义词**：Bootstrapping
- **相关**：Compiler（编译器）、Self-Hosting（自托管）

### Compiler / 编译器
- **中文**：编译器
- **定义**：将高级编程语言源代码转换为低级目标代码（机器码或汇编）的程序
- **说明**：主要阶段：词法分析 → 语法分析 → 语义分析 → 中间代码生成 → 代码优化 → 目标代码生成；代表：GCC、LLVM、Clang
- **相关**：Interpreter（解释器）、Assembler（汇编器）、Linker（链接器）

### Cross Compiler / 交叉编译器
- **中文**：交叉编译器
- **定义**：在 A 平台上运行、生成 B 平台可执行代码的编译器
- **说明**：用于嵌入式开发（在 x86 PC 上编译 ARM 程序）；需要目标平台的工具链
- **相关**：Compiler（编译器）、Target Platform（目标平台）、Toolchain（工具链）

### Front End / 前端
- **中文**：前端
- **定义**：编译器中与源语言相关的部分，负责词法分析、语法分析和语义分析
- **说明**：输出中间表示（IR）；独立于目标机器；包括：Scanner（扫描器）、Parser（解析器）、Type Checker（类型检查器）
- **相关**：Back End（后端）、Lexer（词法分析器）、Parser（语法分析器）

### Intermediate Representation (IR) / 中间表示
- **中文**：中间表示
- **定义**：编译器前端和后端之间的抽象程序表示形式
- **说明**：特点：与源语言和目标机器无关；便于优化和转换；常见形式：三地址码（TAC）、SSA、控制流图（CFG）；LLVM IR 是著名实例
- **同义词**：Intermediate Code（中间代码）
- **相关**：Code Generation（代码生成）、Optimization（优化）、Compiler（编译器）

### Interpreter / 解释器
- **中文**：解释器
- **定义**：直接逐行读取源代码并执行、不生成独立目标程序的程序
- **说明**：启动快、便于调试；执行慢于编译后的程序；代表：Python 解释器、JavaScript 引擎、Ruby；JIT 编译器结合两者优点
- **相关**：Compiler（编译器）、JIT（即时编译）、REPL

### JIT (Just-In-Time) / 即时编译
- **中文**：即时编译
- **定义**：在程序运行时将字节码或中间代码动态编译为机器码的技术
- **说明**：结合解释器灵活性和编译器性能；HotSpot JVM、V8 引擎使用 JIT；可基于运行时信息优化（如内联缓存）
- **相关**：Interpreter（解释器）、Compiler（编译器）、Bytecode（字节码）

### Lexical Analysis / 词法分析
- **中文**：词法分析
- **定义**：将源代码字符流转换为词法单元（Token）序列的过程
- **说明**：编译器第一阶段；识别关键字、标识符、运算符、常量；跳过空白和注释；输出 Token 流给语法分析器
- **同义词**：Scanning（扫描）
- **相关**：Token（词法单元）、Lexer（词法分析器）、Syntax Analysis（语法分析）

### Linker / 链接器
- **中文**：链接器
- **定义**：将多个目标文件和库文件合并为可执行程序的工具
- **说明**：处理符号引用和定义；解析外部符号；地址重定位；静态链接 vs 动态链接
- **相关**：Loader（加载器）、Object File（目标文件）、Symbol（符号）

### Loader / 加载器
- **中文**：加载器
- **定义**：将可执行程序从磁盘加载到内存并准备运行的系统程序
- **说明**：执行重定位、分配内存、解析动态库引用；是操作系统的一部分；加载后跳转到程序入口点
- **相关**：Linker（链接器）、Executable（可执行文件）、Memory Management（内存管理）

### Machine Code / 机器码
- **中文**：机器码
- **定义**：CPU 可直接执行的二进制指令序列
- **说明**：与特定硬件架构绑定；人类难以阅读；是编译器的最终输出；汇编语言是机器码的文本表示
- **相关**：Assembly Language（汇编语言）、Instruction（指令）、Binary（二进制）

### Macro / 宏
- **中文**：宏
- **定义**：在编译前通过文本替换进行扩展的代码模板
- **说明**：预处理阶段处理；C/C++ 的 #define；可带参数；可能引入副作用；与函数不同，宏在编译前展开
- **相关**：Preprocessor（预处理器）、Template（模板）、Metaprogramming（元编程）

### Object File / 目标文件
- **中文**：目标文件
- **定义**：编译器生成的包含机器码和符号表、但尚未链接的半成品文件
- **说明**：格式：ELF（Linux）、PE（Windows）、Mach-O（macOS）；包含代码段、数据段、符号表、重定位信息
- **相关**：Compiler（编译器）、Linker（链接器）、Executable（可执行文件）

### Optimization / 优化
- **中文**：优化
- **定义**：在保持程序语义不变的前提下改进代码性能（速度、空间、功耗）的变换
- **说明**：级别：机器无关优化（IR 层）、机器相关优化（后端）；技术：常量折叠、死代码消除、循环优化、内联展开
- **相关**：Compiler（编译器）、IR（中间表示）、Code Generation（代码生成）

### Parser / 解析器
- **中文**：解析器
- **定义**：根据语法规则将 Token 流构建为抽象语法树（AST）的程序
- **说明**：编译器核心组件；分为自顶向下（LL）和自底向上（LR）两类；检测语法错误
- **同义词**：Syntax Analyzer（语法分析器）
- **相关**：AST（抽象语法树）、Grammar（文法）、Token（词法单元）

### Preprocessor / 预处理器
- **中文**：预处理器
- **定义**：在编译前对源代码进行文本处理的程序
- **说明**：功能：宏展开、文件包含（#include）、条件编译（#ifdef）；C/C++ 预处理器是独立阶段；处理后的代码送给编译器
- **相关**：Macro（宏）、Conditional Compilation（条件编译）、Compilation（编译）

### Semantic Analysis / 语义分析
- **中文**：语义分析
- **定义**：检查程序是否有意义（类型一致性、变量声明、作用域等）的编译阶段
- **说明**：在语法正确基础上检查语义；包括：类型检查、符号表管理、作用域分析；发现：未声明变量、类型不匹配
- **相关**：Type Checking（类型检查）、Symbol Table（符号表）、Scope（作用域）

### Source Code / 源代码
- **中文**：源代码
- **定义**：程序员用高级编程语言编写的、人类可读的程序文本
- **说明**：是编译器的输入；需遵循语言语法和语义规则；通过编译/解释转换为可执行程序
- **相关**：Compiler（编译器）、Programming Language（编程语言）、Target Code（目标代码）

### Symbol Table / 符号表
- **中文**：符号表
- **定义**：编译器中存储标识符（变量、函数、类）信息的数据结构
- **说明**：记录：名称、类型、作用域、内存地址、行号等；贯穿编译全过程；支持快速查找和作用域管理
- **相关**：Semantic Analysis（语义分析）、Scope（作用域）、Identifier（标识符）

### Syntax Analysis / 语法分析
- **中文**：语法分析
- **定义**：根据文法规则检查 Token 序列是否构成合法句子并构建语法树的过程
- **说明**：编译器第二阶段；识别程序结构；输出抽象语法树（AST）或分析树；检测语法错误
- **同义词**：Parsing（解析）
- **相关**：Parser（解析器）、Grammar（文法）、AST（抽象语法树）

### Target Code / 目标代码
- **中文**：目标代码
- **定义**：编译器生成的、可在目标机器上运行的代码（机器码或汇编）
- **说明**：目标代码尚未链接；需经过链接器处理才能成为可执行文件；优化后的目标代码效率更高
- **相关**：Machine Code（机器码）、Code Generation（代码生成）、Linker（链接器）

### Token / 词法单元
- **中文**：词法单元
- **定义**：词法分析产生的、具有独立语义的最小语法单位
- **说明**：类型：关键字（Keyword）、标识符（Identifier）、运算符（Operator）、常量（Literal）、分隔符；是语法分析的输入
- **同义词**：Lexeme（词素）
- **相关**：Lexical Analysis（词法分析）、Lexer（词法分析器）、Grammar（文法）

### Toolchain / 工具链
- **中文**：工具链
- **定义**：用于编译、链接、调试程序的完整工具集合
- **说明**：通常包括：预处理器、编译器、汇编器、链接器、调试器、库管理器；GNU Toolchain：GCC、Binutils、GDB、Glibc
- **相关**：Compiler（编译器）、Linker（链接器）、Debugger（调试器）

### Translation Unit / 翻译单元
- **中文**：翻译单元
- **定义**：经过预处理后、送给编译器进行编译的单个源代码文件
- **说明**：.c/.cpp 文件预处理后的结果；包含所有 #include 的内容；是编译的基本单位
- **相关**：Preprocessor（预处理器）、Source File（源文件）、Compilation（编译）

### Virtual Machine / 虚拟机
- **中文**：虚拟机
- **定义**：模拟计算机硬件、执行中间代码（字节码）的软件环境
- **说明**：提供平台无关的执行环境；代表：JVM（Java 虚拟机）、CLR（.NET 公共语言运行时）、WebAssembly 虚拟机
- **相关**：Bytecode（字节码）、Interpreter（解释器）、JIT（即时编译）

### Yacc / Yacc
- **中文**：Yet Another Compiler Compiler
- **定义**：经典的语法分析器生成器，根据文法描述自动生成 LALR 解析器
- **说明**：输入：上下文无关文法；输出：C 语言解析器代码；GNU 版本：Bison；常与 Lex/Flex 配合使用
- **相关**：Lex/Flex、Parser Generator（解析器生成器）、LALR
