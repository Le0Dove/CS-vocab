# 01. 基础概念 (Basic Concepts)
### Backbone / 骨干网
- **音标**：/ˈbækboʊn/
- **中文**：骨干网
- **定义**：连接网络核心节点的高带宽主干网络
- **说明**：骨干网承载大量数据传输；ISP 骨干网连接各地区网络；典型速率 100Gbps+
- **相关**：ISP（互联网服务提供商）、Core Network（核心网）
### Bandwidth / 带宽
- **音标**：/ˈbændwɪdθ/
- **中文**：带宽
- **定义**：通信链路在单位时间内能传输的最大数据量，单位为 bps
- **说明**：带宽和吞吐量不同：带宽是理论最大值，吞吐量是实际传输速率；瓶颈带宽（Bottleneck Bandwidth）决定端到端最大速率
- **同义词**：Bandwidth、Throughput（吞吐量，不同概念）
- **相关**：Latency（延迟）、Throughput（吞吐量）、bps
### Bandwidth-Delay Product / 带宽延迟积
- **中文**：带宽延迟积
- **定义**：带宽与往返延迟的乘积，表示管道中能容纳的最大比特数
- **说明**：BDP = Bandwidth × RTT；用于确定 TCP 缓冲区大小；BDP 越大，链路利用率越高所需的窗口越大
- **同义词**：BDP
- **相关**：Bandwidth（带宽）、RTT（往返时间）、TCP Window（TCP 窗口）
### bps (bits per second) / 比特每秒
- **中文**：比特每秒
- **定义**：数据传输速率的单位
- **说明**：常见：Kbps (10³)、Mbps (10⁶)、Gbps (10⁹)、Tbps (10¹²)；注意区分 bps 和 Bps（字节每秒，1 Bps = 8 bps）
- **相关**：Bandwidth（带宽）、Throughput（吞吐量）
### Broadcast / 广播
- **音标**：/ˈbrɔːdkæst/
- **中文**：广播
- **定义**：将数据发送给网络中所有节点的一对多通信方式
- **说明**：广播地址通常为全 1（如 255.255.255.255）；二层广播（MAC 全 F）和三广播（IP 广播）；广播域（Broadcast Domain）过大影响性能
- **相关**：Multicast（多播）、Unicast（单播）、Broadcast Domain（广播域）
### Client / 客户端
- **音标**：/ˈklaɪənt/
- **中文**：客户端
- **定义**：向服务器请求服务的网络端点
- **说明**：客户端-服务器架构中，客户端发起连接；浏览器、手机 App、SSH 客户端都是客户端
- **相关**：Server（服务器）、Client-Server Model（客户端-服务器模型）
### Client-Server Model / 客户端-服务器模型
- **中文**：客户端-服务器模型
- **定义**：客户端发送请求、服务器处理请求并返回响应的分布式架构模式
- **说明**：服务器持续运行监听端口，客户端按需连接；是最常见的网络应用架构
- **相关**：P2P（点对点）、Client（客户端）、Server（服务器）
### Connection / 连接
- **音标**：/kəˈnekʃn/
- **中文**：连接
- **定义**：两点之间建立的逻辑通信信道，通常需要握手确认
- **说明**：TCP 是面向连接的协议（三次握手建立连接）；UDP 是无连接的；连接有状态，需要维护连接表
- **相关**：TCP、Connection-Oriented（面向连接）、Connectionless（无连接）
### Datagram / 数据报
- **音标**：/ˈdeɪtəɡræm/
- **中文**：数据报
- **定义**：独立传输的完整数据单元，包含目的地地址，不需要预先建立连接
- **说明**：UDP 使用数据报；每个数据报独立路由；不保证送达、顺序，但开销小
- **相关**：Packet（数据包）、UDP、Connectionless（无连接）
### Encapsulation / 封装
- **音标**：/ɪnˈkæpsjuˈleɪʃn/
- **中文**：封装
- **定义**：每层协议将上层数据打包，添加本层头部/尾部形成新 PDU 的过程
- **说明**：应用层数据 → 传输层加 TCP/UDP 头 → 网络层加 IP 头 → 数据链路层加 MAC 头/尾；层层封装形成洋葱结构
- **相关**：PDU（协议数据单元）、Protocol Stack（协议栈）、Header（头部）
### End-to-End Principle / 端到端原则
- **中文**：端到端原则
- **定义**：网络设计原则，尽可能将功能放在通信端点而非中间节点
- **说明**：网络尽保持简单（尽量只进行路由，提交应用逻辑给端点）；TCP 可靠性在端实现而非路由器
- **相关**：TCP、Protocol Stack（协议栈）
### Full Duplex / 全双工
- **音标**：/fʊl ˈdjuːplɛks/
- **中文**：全双工
- **定义**：通信双方可同时发送和接收数据的通信模式
- **说明**：以太网交换机、TCP 连接都是全双工；对比半双工（Half Duplex）只能交替通信；对比单工（Simplex）只能单向通信
- **相关**：Half Duplex（半双工）、Simplex（单工）、Duplex
### Gateway / 网关
- **音标**：/ˈɡeɪtweɪ/
- **中文**：网关
- **定义**：连接不同网络、在不同协议之间转发数据的中介设备
- **说明**：默认网关（Default Gateway）是数据包出网的出口；网关工作在 OSI 模型多个层次
- **同义词**：Default Gateway（默认网关）
- **相关**：Router（路由器）、NAT（网络地址转换）
### Half Duplex / 半双工
- **中文**：半双工
- **定义**：通信双方可发送和接收但不能同时进行的通信模式
- **说明**：传统集线器（Hub）、Wi-Fi 是半双工；需要 CSMA/CD 或 CSMA/CA 协调
- **相关**：Full Duplex（全双工）、CSMA/CD（载波监听多路访问/冲突检测）
### Hop / 跳
- **音标**：/hɒp/
- **中文**：跳
- **定义**：数据包经过一个路由器从源到目的的一步
- **说明**：TTL（生存时间）每经过一跳减 1；跳数用于路由协议（如 RIP 以跳数度量距离）
- **相关**：TTL（生存时间）、Router（路由器）、Hop Count（跳数）
### Host / 主机
- **音标**：/hoʊst/
- **中文**：主机
- **定义**：网络中拥有 IP 地址的终端设备
- **说明**：服务器、PC、手机、IoT 设备都是主机；每台主机有唯一 IP 地址；主机名（Hostname）是主机的可读名称
- **相关**：IP Address（IP 地址）、Server（服务器）、Client（客户端）
### Internet / 互联网
- **音标**：/ˈɪntərnet/
- **中文**：互联网
- **定义**：全球最大的、由无数网络互联而成的计算机网络
- **说明**：Internet（大写 I）特指全球互联网，internet（小写）泛指互联网络；基于 TCP/IP 协议族；由 IANA/ICANN 管理
- **相关**：Intranet（内联网）、Extranet（外联网）、TCP/IP
### Intranet / 内联网
- **音标**：/ˈɪntrənet/
- **中文**：内联网
- **定义**：组织内部使用互联网技术搭建的私有网络
- **说明**：使用 TCP/IP 协议，但与外网隔离；通过防火墙和 NAT 访问外网
- **相关**：Internet（互联网）、Extranet（外联网）
### Jitter / 抖动
- **音标**：/ˈdʒɪtər/
- **中文**：抖动
- **定义**：数据包到达时间的变化（延迟的方差）
- **说明**：抖动对实时应用（VoIP、视频通话）影响大；接收方用抖动缓冲区（Jitter Buffer）平滑抖动
- **相关**：Latency（延迟）、Packet Loss（丢包）、VoIP
### LAN (Local Area Network) / 局域网
- **发音**：/læn/
- **中文**：局域网
- **定义**：覆盖有限地理范围（办公室、校园）的计算机网络
- **说明**：LAN 由同一组织管理，通常使用以太网；速度快（1Gbps+）、延迟低
- **相关**：WAN（广域网）、MAN（城域网）、Ethernet（以太网）
### Latency / 延迟
- **音标**：/ˈleɪtənsi/
- **中文**：延迟（时延）
- **定义**：数据从源到目的地所需的时间
- **说明**：延迟 = 传播延迟 + 传输延迟 + 处理延迟 + 排队延迟；应用关注 RTT（往返时间）
- **同义词**：Delay（时延）
- **相关**：RTT（往返时间）、Bandwidth（带宽）、Jitter（抖动）
### MAN (Metropolitan Area Network) / 城域网
- **发音**：/mæn/
- **中文**：城域网
- **定义**：覆盖城市范围的计算机网络，介于 LAN 和 WAN 之间
- **相关**：LAN（局域网）、WAN（广域网）
### Multicast / 多播
- **中文**：多播
- **定义**：将数据发送给特定组中多个节点的一对多通信方式
- **说明**：IP 多播地址 224.0.0.0-239.255.255.255；IGMP 管理多播组成员；用于视频会议、流媒体、IPTV
- **相关**：Unicast（单播）、Broadcast（广播）、IGMP
### Network / 网络
- **中文**：网络
- **定义**：由通信链路连接的节点集合，用于数据交换
- **说明**：网络分类：LAN、WAN、MAN、PAN 等；分层架构（OSI/TCP-IP）简化网络设计
- **相关**：Node（节点）、Link（链路）、Protocol（协议）
### Network Protocol / 网络协议
- **中文**：网络协议
- **定义**：规定网络中数据通信的格式、时序和错误处理的一组规则
- **说明**：协议三要素：语法（格式）、语义（含义）、时序（顺序）；HTTP、TCP/IP、DNS 是常见协议；RFC 文档标准化协议
- **相关**：RFC、Protocol Stack（协议栈）、TCP/IP
### Node / 节点
- **音标**：/noʊd/
- **中文**：节点
- **定义**：网络中的连接设备，可以是主机、路由器、交换机等
- **说明**：末端节点（End Node）生成/接收数据；中间节点（Intermediate Node）转发数据
- **相关**：Host（主机）、Router（路由器）、Switch（交换机）
### OSI Model / OSI 模型
- **发音**：/oʊ-ɛs-aɪ ˈmɒdl/
- **中文**：开放系统互连模型
- **定义**：ISO 制定的七层网络参考模型，标准委员会用于描述和理解网络通信
- **说明**：七层：物理层 → 数据链路层 → 网络层 → 传输层 → 会话层 → 表示层 → 应用层
- **相关**：TCP/IP Model（TCP/IP 模型）、Protocol Stack（协议栈）
### Packet / 数据包
- **中文**：数据包
- **定义**：网络层的数据传输单元（PDU），包含 IP 头部和数据
- **说明**：数据包可以分片（Fragmentation）以适应 MTU；包交换（Packet Switching）是现代网络的基础
- **相关**：Frame（帧）、Datagram（数据报）、Packet Switching（包交换）
### Packet Switching / 包交换
- **中文**：包交换
- **定义**：将数据分割为独立数据包分别传输，每个包独立路由的交换技术
- **说明**：与电路交换（Circuit Switching）相对；统计复用效率高；IP 网络基于包交换；分为数据报服务和无连接方式
- **相关**：Circuit Switching（电路交换）、Packet（数据包）、Router（路由器）
### Peer-to-Peer (P2P) / 点对点
- **发音**：/piːr-tuː-piːr/
- **中文**：点对点
- **定义**：每个节点既充当客户端又充当服务器的分布式架构
- **说明**：P2P 无需中心服务器；BitTorrent、Skype 早期版、区块链使用 P2P；具有自组织性和扩展性
- **相关**：Client-Server Model、Distributed System（分布式系统）
### Protocol Stack / 协议栈
- **中文**：协议栈
- **定义**：按层次组织的网络协议集合，每层提供特定服务
- **说明**：TCP/IP 协议栈四层：应用层、传输层、网络层、网络接口层；每层为上层提供服务，使用下层服务
- **相关**：OSI Model（OSI 模型）、TCP/IP、Encapsulation（封装）
### RFC (Request for Comments) / 请求评论
- **发音**：/ɑːr-ɛf-siː/
- **中文**：请求评论
- **定义**：IETF 发布的技术文档，定义互联网标准和协议
- **说明**：RFC 有编号（如 RFC 793 定义 TCP）；不是所有 RFC 都是标准（有实验性、信息性等类别）
- **相关**：IETF（互联网工程任务组）、Protocol（协议）、Standard（标准）
### Round-Trip Time (RTT) / 往返时间
- **音标**：/raʊnd trɪp taɪm/
- **中文**：往返时间
- **定义**：数据包从发送端到接收端再返回确认所需的总时间
- **说明**：RTT 影响 TCP 吞吐量；ping 命令测量 RTT；卫星链路 RTT 大，局域网 RTT 小
- **相关**：Latency（延迟）、TCP、Bandwidth
### Segment / 段
- **中文**：段
- **定义**：传输层的数据传输单元，TCP 的数据包称为 TCP Segment，UDP 称为 UDP Datagram
- **说明**：TCP Segment 包含 TCP 头部和载荷；分段与重组是传输层功能
- **相关**：TCP、Packet（数据包）、Frame（帧）
### Server / 服务器
- **中文**：服务器
- **定义**：为客户端提供服务的网络端点
- **说明**：服务器持续运行，被动接收客户端请求；物理服务器和软件服务器两个含义
- **相关**：Client（客户端）、Client-Server Model
### Simplex / 单工
- **中文**：单工
- **定义**：只能单向通信的模式，一方只能发送另一方只能接收
- **说明**：广播电台、电视是单工通信；对比全双工和半双工
- **相关**：Full Duplex（全双工）、Half Duplex（半双工）
### TCP/IP Model / TCP/IP 模型
- **中文**：TCP/IP 模型
- **定义**：互联网使用的四层协议模型，实际标准
- **说明**：四层：应用层、传输层、网络层、网络接口层；比 OSI 简化，是互联网的事实标准
- **相关**：OSI Model（OSI 模型）、Protocol Stack（协议栈）
### Throughput / 吞吐量
- **中文**：吞吐量
- **定义**：实际单位时间内成功传输的数据量，通常小于带宽
- **说明**：吞吐量受带宽、延迟、丢包、拥塞控制等影响；TCP 吞吐量 ≈ Window/ RTT
- **同义词**：ThroughputRate
- **相关**：Bandwidth（带宽）、Latency（延迟）、TCP
### Unicast / 单播
- **中文**：单播
- **定义**：将数据发送给单一特定目标节点的一对一通信方式
- **说明**：单播是最常见的通信方式；每次传输只涉及一个发送方和一个接收方
- **相关**：Multicast（多播）、Broadcast（广播）
### WAN (Wide Area Network) / 广域网
- **中文**：广域网
- **定义**：覆盖广大地理范围的计算机网络，连接多个 LAN
- **说明**：互联网是最大的 WAN；WAN 技术：租用线、MPLS、SD-WAN、卫星；通常由 ISP 运营
- **相关**：LAN（局域网）、MAN（城域网）
