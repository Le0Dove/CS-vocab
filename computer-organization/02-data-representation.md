# 02. 数据表示与编码 (Data Representation & Encoding)
### ASCII / 美国信息交换标准码
- **中文**：美国信息交换标准码
- **定义**：7 位（或 8 位扩展）编码标准，共 128 个字符
- **说明**：编码英文字母、数字、符号等字符
- **相关**：UTF-8、Encoding（编码）
### Complementary Code / 补码
- **中文**：补码
- **定义**：用于表示有符号整数并且简化加减法运算的二进制编码方式
- **说明**：补码 = 反码（按位取反）后加 1
- **相关**：Two's Complement（同义词）、Binary（二进制）、Overflow（溢出）
### Fixed-Point / 定点数
- **中文**：定点数
- **定义**：小数点在数的表示中位置固定不变的数
- **说明**：定点数运算速度快、实现简单，但能表示的范围和精度受限
- **相关**：Floating-Point（浮点数）、Binary（二进制）
### Floating-Point / 浮点数
- **中文**：浮点数
- **定义**：用科学记数法形式（尾数、基数、指数）表示实数的编码方式，小数点位置可"浮动"
- **说明**：IEEE 754 定义单精度（32 位）和双精度（64 位）
- **相关**：IEEE 754、Fixed-Point（定点数）
### Gray Code / 格雷码
- **中文**：格雷码
- **定义**：相邻编码只有一位不同的可靠性编码
- **说明**：格雷码用于旋转编码器、卡诺图、异步 FIFO 等场景
- **相关**：Binary（二进制）
### Hexadecimal / 十六进制
- **中文**：十六进制
- **定义**：以 16 为基数的计数系统，使用 0-9 和 A-F
- **说明**：十六进制一位对应 4 个二进制位，方便读取和书写
- **相关**：Binary（二进制）
### IEEE 754 / IEEE 浮点数标准
- **中文**：IEEE 754
- **定义**：IEEE 制定的浮点数运算标准
- **说明**：定义单精度（32 位）和双精度（64 位）浮点数格式
- **相关**：Floating-Point（浮点数）
### Octal / 八进制
- **中文**：八进制
- **定义**：以 8 为基数的计数系统，使用 0-7 共 8 个数字
- **说明**：一位八进制对应 3 个二进制位
- **相关**：Binary（二进制）
### One's Complement / 反码
- **中文**：反码
- **定义**：将有符号整数的所有位取反而得到的编码方式
- **说明**：反码存在正负零问题，现代计算机采用补码替代反码
- **相关**：Two's Complement（补码）
### Overflow / 溢出
- **中文**：溢出
- **定义**：运算结果超过用给定位数所能表示的数据范围
- **说明**：溢出导致结果错误或触发异常处理
- **相关**：Carry（进位）、Two's Complement（补码）
### Parity Bit / 奇偶校验位
- **中文**：奇偶校验位
- **定义**：附加在数据位上用于错误检测的校验位
- **说明**：分为奇校验和偶校验
- **相关**：Error Detection（错误检测）
### Sign Bit / 符号位
- **中文**：符号位
- **定义**：二进制数中用于表示正负号的最高位（0 正 1 负）
- **相关**：Two's Complement（补码）、Binary（二进制）
### Sign Extension / 符号扩展
- **中文**：符号扩展
- **定义**：将有符号数从较小位数扩展到较大位数时复制符号位的操作
- **相关**：Two's Complement（补码）
### Signed Magnitude / 原码
- **中文**：原码
- **定义**：最高位为符号位，其余位表示绝对值的编码方式
- **说明**：原码直观容易理解，但存在正负零且加减运算不统一
- **相关**：Two's Complement（补码）
### Two's Complement / 补码
- **中文**：补码
- **定义**：同 Complementary Code
- **相关**：Complementary Code（补码）
### Unicode / 统一码
- **中文**：统一码
- **定义**：为全世界每种文字系统每个字符分配唯一编码的国际标准
- **说明**：与 UTF-8/UTF-16/UTF-32 不同的编码方式
- **相关**：UTF-8、ASCII
### UTF-8
- **中文**：Unicode 转换格式-8
- **定义**：Unicode 最常用的可变长度字符编码，向后兼容 ASCII
- **说明**：UTF-8 每个字符占 1-4 个字节；互联网和现代系统默认文本编码
- **相关**：Unicode（统一码）、ASCII
