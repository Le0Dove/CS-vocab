# 03. 树结构 (Tree Structures)

### AVL Tree / AVL 树
- **音标**：/ˌeɪ viː ˈel triː/
- **中文**：AVL 树
- **定义**：一种自平衡二叉搜索树，任何节点的左右子树高度差不超过 1
- **说明**：AVL 树由 Adelson-Velsky 和 Landis 发明；通过旋转（Rotation）保持平衡；查找、插入、删除都是 O(log n)；比红黑树更严格平衡，查找更快但插入/删除旋转更多
- **相关**：Balanced Binary Tree（平衡二叉树）、Red-Black Tree（红黑树）、Rotation（旋转）

### B-Tree / B 树
- **音标**：/biː triː/
- **中文**：B 树
- **定义**：一种自平衡多路搜索树，节点可包含多个键和子节点，广泛用于数据库和文件系统
- **说明**：B 树的阶（order）m 表示每个节点最多 m 个子节点；所有叶子节点在同一层；数据库索引（如 MySQL InnoDB）使用 B+ 树（B 树的变种）
- **相关**：B+ Tree（B+ 树）、Database Index（数据库索引）、Balanced Tree（平衡树）

### B+ Tree / B+ 树
- **音标**：/biː plʌs triː/
- **中文**：B+ 树
- **定义**：B 树的变种，非叶子节点只存键不存数据，所有数据存储在叶子节点并通过链表连接
- **说明**：B+ 树更适合范围查询和顺序遍历；叶子节点形成有序链表；MySQL、MongoDB 等数据库使用 B+ 树作为索引结构
- **相关**：B-Tree（B 树）、Database Index（数据库索引）

### Balanced Binary Tree / 平衡二叉树
- **音标**：/ˈbælənst ˈbaɪnəri triː/
- **中文**：平衡二叉树
- **定义**：左右子树高度差不超过某个阈值（通常为 1）的二叉树，保证操作效率
- **说明**：平衡二叉树避免退化到 O(n) 的链表形态；常见实现：AVL 树、红黑树、Treap；插入/删除/查找都是 O(log n)
- **同义词**：Height-Balanced Tree（高度平衡树）
- **相关**：AVL Tree（AVL 树）、Red-Black Tree（红黑树）、Binary Search Tree（二叉搜索树）

### Binary Heap / 二叉堆
- **音标**：/ˈbaɪnəri hiːp/
- **中文**：二叉堆
- **定义**：一种完全二叉树，满足堆性质（父节点键值大于/小于子节点）
- **说明**：分为最大堆（Max Heap）和最小堆（Min Heap）；用数组实现：父节点 i 的左右子节点为 2i+1 和 2i+2；插入和删除 O(log n)，查找最值 O(1)
- **相关**：Heap（堆）、Priority Queue（优先队列）、Complete Binary Tree（完全二叉树）

### Binary Search Tree (BST) / 二叉搜索树
- **音标**：/ˈbaɪnəri sɜːrtʃ triː/
- **中文**：二叉搜索树
- **定义**：满足左子树所有节点 < 根节点 < 右子树所有节点的二叉树
- **说明**：BST 支持高效查找、插入、删除（平均 O(log n)）；但不平衡时（如顺序插入）退化为 O(n)；因此需要自平衡 BST（AVL、红黑树）
- **同义词**：BST
- **相关**：Balanced Binary Tree（平衡二叉树）、AVL Tree（AVL 树）、Inorder Traversal（中序遍历）
- **例句**：In a BST, inorder traversal yields sorted elements.

### Binary Tree / 二叉树
- **音标**：/ˈbaɪnəri triː/
- **中文**：二叉树
- **定义**：每个节点最多有两个子节点（左子节点和右子节点）的树结构
- **说明**：二叉树是树结构的基础；常见类型：满二叉树、完全二叉树、二叉搜索树、平衡二叉树；遍历方式：前序、中序、后序、层序
- **相关**：Tree（树）、Node（节点）、Traversal（遍历）

### Complete Binary Tree / 完全二叉树
- **音标**：/kəmˈpliːt ˈbaɪnəri triː/
- **中文**：完全二叉树
- **定义**：除最后一层外所有层都满，且最后一层节点从左到右连续排列的二叉树
- **说明**：完全二叉树可以用数组紧凑存储（无空间浪费）；堆就是完全二叉树；n 个节点的完全二叉树高度为 ⌊log₂n⌋+1
- **相关**：Full Binary Tree（满二叉树）、Binary Heap（二叉堆）

### Depth / 深度
- **音标**：/depθ/
- **中文**：深度
- **定义**：从根节点到某节点的路径上的边数
- **说明**：根节点的深度为 0（或 1，取决于定义）；树的最大深度即为树的高度
- **相关**：Height（高度）、Level（层）、Tree（树）

### Full Binary Tree / 满二叉树
- **音标**：/fʊl ˈbaɪnəri triː/
- **中文**：满二叉树
- **定义**：每个非叶子节点都有两个子节点，且所有叶子节点都在同一层的二叉树
- **说明**：满二叉树是完全二叉树的特例；高度为 h 的满二叉树有 2ʰ-1 个节点
- **相关**：Complete Binary Tree（完全二叉树）、Binary Tree（二叉树）

### Heap / 堆
- **音标**：/hiːp/
- **中文**：堆
- **定义**：满足堆性质的完全二叉树，用于实现优先队列
- **说明**：最大堆：父节点 ≥ 子节点；最小堆：父节点 ≤ 子节点；常用数组实现；堆排序使用堆结构；Dijkstra 和 Prim 算法用最小堆优化
- **相关**：Binary Heap（二叉堆）、Priority Queue（优先队列）、Heap Sort（堆排序）

### Height / 高度
- **音标**：/haɪt/
- **中文**：高度
- **定义**：从某节点到最远叶子节点的路径上的边数
- **说明**：树的高度 = 根节点的高度；空树高度为 -1（或 0）；叶子节点高度为 0；平衡树要求高度差不超过阈值
- **相关**：Depth（深度）、Balanced Tree（平衡树）

### Inorder Traversal / 中序遍历
- **音标**：/ˈɪnɔːrdər trəˈvɜːrsəl/
- **中文**：中序遍历
- **定义**：二叉树遍历方式：左子树 -> 根节点 -> 右子树
- **说明**：BST 的中序遍历结果是有序序列；递归实现最简单；迭代实现用栈；Morris 遍历可实现 O(1) 空间
- **相关**：Preorder Traversal（前序遍历）、Postorder Traversal（后序遍历）、BST（二叉搜索树）

### Leaf / 叶子
- **音标**：/liːf/
- **中文**：叶子
- **定义**：没有子节点的树节点
- **说明**：叶子节点位于树的最底层；满二叉树的叶子节点数 = 内部节点数 + 1
- **同义词**：Leaf Node（叶子节点）
- **相关**：Internal Node（内部节点）、Tree（树）

### Left Child / 左子节点
- **音标**：/left tʃaɪld/
- **中文**：左子节点
- **定义**：二叉树中位于父节点左侧的子节点
- **说明**：在 BST 中，左子节点的值小于父节点；用数组存储时，父节点 i 的左子节点为 2i+1
- **相关**：Right Child（右子节点）、Parent（父节点）

### Level / 层
- **音标**：/ˈlevl/
- **中文**：层
- **定义**：树中深度相同的节点集合
- **说明**：根节点在第 0 层（或第 1 层）；树的层数 = 高度 + 1；层序遍历（Level Order Traversal）按层访问节点
- **相关**：Depth（深度）、Height（高度）

### Level Order Traversal / 层序遍历
- **音标**：/ˈlevl ˈɔːrdər trəˈvɜːrsəl/
- **中文**：层序遍历
- **定义**：按树的层次从上到下、从左到右依次访问节点的遍历方式
- **说明**：层序遍历使用队列实现（BFS）；用于求树高、判断完全二叉树、连接同层节点
- **同义词**：Breadth-First Traversal（广度优先遍历）
- **相关**：BFS、Queue（队列）、Tree（树）

### Max Heap / 最大堆
- **音标**：/mæks hiːp/
- **中文**：最大堆
- **定义**：父节点的键值大于或等于所有子节点的堆
- **说明**：最大堆的堆顶是最大值；extract_max 操作移除并返回最大值；用于堆排序、Top-K 问题
- **相关**：Min Heap（最小堆）、Heap（堆）、Priority Queue（优先队列）

### Min Heap / 最小堆
- **音标**：/mɪn hiːp/
- **中文**：最小堆
- **定义**：父节点的键值小于或等于所有子节点的堆
- **说明**：最小堆的堆顶是最小值；extract_min 操作移除并返回最小值；用于 Dijkstra、Prim、Huffman 编码
- **相关**：Max Heap（最大堆）、Heap（堆）

### N-ary Tree / N 叉树
- **音标**：/ɛn ˈɛri triː/
- **中文**：N 叉树
- **定义**：每个节点最多有 N 个子节点的树
- **说明**：二叉树是 N=2 的特例；文件系统目录树通常是 N 叉树；N 叉树可用"左孩子右兄弟"表示法转为二叉树
- **相关**：Binary Tree（二叉树）、Tree（树）

### Parent / 父节点
- **音标**：/ˈperənt/
- **中文**：父节点
- **定义**：树中某个节点的直接上级节点
- **说明**：根节点没有父节点；除根外每个节点有且只有一个父节点
- **相关**：Child（子节点）、Root（根节点）、Tree（树）

### Path / 路径
- **音标**：/pæθ/
- **中文**：路径
- **定义**：树中从一个节点到另一个节点经过的节点和边的序列
- **说明**：树中任意两节点之间有且只有一条路径；路径长度 = 边的数量
- **相关**：Edge（边）、Node（节点）、Tree（树）

### Postorder Traversal / 后序遍历
- **音标**：/ˈpoʊstɔːrdər trəˈvɜːrsəl/
- **中文**：后序遍历
- **定义**：二叉树遍历方式：左子树 -> 右子树 -> 根节点
- **说明**：后序遍历在访问根节点前先访问左右子树；用于删除树（先删子节点再删根）、计算目录大小、表达式求值（后缀表达式）
- **相关**：Inorder Traversal（中序遍历）、Preorder Traversal（前序遍历）

### Preorder Traversal / 前序遍历
- **音标**：/ˈpriːɔːrdər trəˈvɜːrsəl/
- **中文**：前序遍历
- **定义**：二叉树遍历方式：根节点 -> 左子树 -> 右子树
- **说明**：前序遍历先访问根节点；用于复制树、序列化树、文件系统遍历；根节点总在子节点之前
- **相关**：Inorder Traversal（中序遍历）、Postorder Traversal（后序遍历）

### Red-Black Tree / 红黑树
- **音标**：/rɛd blæk triː/
- **中文**：红黑树
- **定义**：一种自平衡二叉搜索树，通过节点颜色（红/黑）约束保持近似平衡
- **说明**：红黑树性质：根黑、叶黑（NIL）、红节点子必黑、黑高相同；插入/删除最多两次旋转；C++ map/set、Java TreeMap、Linux 内核用红黑树；比 AVL 树旋转少，插入/删除更快
- **相关**：AVL Tree（AVL 树）、Balanced BST（平衡二叉搜索树）、Rotation（旋转）
- **例句**：Java's TreeMap is implemented using a red-black tree.

### Right Child / 右子节点
- **音标**：/raɪt tʃaɪld/
- **中文**：右子节点
- **定义**：二叉树中位于父节点右侧的子节点
- **说明**：在 BST 中，右子节点的值大于父节点；用数组存储时，父节点 i 的右子节点为 2i+2
- **相关**：Left Child（左子节点）、Parent（父节点）

### Root / 根节点
- **音标**：/ruːt/
- **中文**：根节点
- **定义**：树的顶层节点，没有父节点的节点
- **说明**：一棵树只有一个根节点；所有其他节点都可以通过从根出发的路径到达
- **相关**：Parent（父节点）、Leaf（叶子）、Tree（树）

### Rotation / 旋转
- **音标**：/roʊˈteɪʃn/
- **中文**：旋转
- **定义**：平衡二叉树中调整节点位置以保持平衡的局部重构操作
- **说明**：基本旋转：左旋（Left Rotation）和右旋（Right Rotation）；组合旋转：左右旋、右左旋；AVL 树和红黑树通过旋转保持平衡；旋转不改变中序遍历结果
- **相关**：AVL Tree（AVL 树）、Red-Black Tree（红黑树）、Balanced Tree（平衡树）

### Segment Tree / 线段树
- **音标**：/ˈseɡmənt triː/
- **中文**：线段树
- **定义**：一种二叉树数据结构，用于高效处理区间查询和修改
- **说明**：线段树每个节点代表一个区间；支持区间和、区间最值、区间更新等操作；查询和更新 O(log n)；常用于数组区间问题；可扩展为懒惰传播（Lazy Propagation）处理区间更新
- **相关**：Interval Tree（区间树）、Binary Indexed Tree（树状数组）、Range Query（区间查询）

### Sibling / 兄弟节点
- **音标**：/ˈsɪblɪŋ/
- **中文**：兄弟节点
- **定义**：具有相同父节点的节点
- **说明**：二叉树中最多有两个兄弟（左兄弟和右兄弟）；N 叉树中可以有多个兄弟
- **相关**：Parent（父节点）、Child（子节点）

### Subtree / 子树
- **音标**：/ˈsʌbtriː/
- **中文**：子树
- **定义**：以树中某个节点为根的树，包括该节点及其所有后代
- **说明**：每个节点都是一棵子树的根；空树是任何节点的子树；子树大小 = 该节点及其后代节点数
- **相关**：Tree（树）、Root（根节点）

### Threaded Binary Tree / 线索二叉树
- **音标**：/ˈθrɛdɪd ˈbaɪnəri triː/
- **中文**：线索二叉树
- **定义**：利用空指针域存储遍历前驱/后继信息的二叉树
- **说明**：线索化后可以不使用递归或栈进行中序遍历；分为单向线索和双向线索；节省空间但增加实现复杂度
- **相关**：Binary Tree（二叉树）、Traversal（遍历）

### Tree / 树
- **音标**：/triː/
- **中文**：树
- **定义**：由节点和边组成的层次结构，任意两节点间有且仅有一条路径的无环连通图
- **说明**：树的基本术语：根、叶子、父节点、子节点、兄弟、深度、高度、路径；树的遍历：前序、中序、后序、层序；应用：文件系统、HTML DOM、决策树、表达式解析
- **相关**：Graph（图）、Binary Tree（二叉树）、Forest（森林）

### Trie / 前缀树
- **音标**：/traɪ/
- **中文**：前缀树
- **定义**：一种多叉树，用于高效存储和检索字符串集合，每条边代表一个字符
- **说明**：Trie 支持 O(m) 时间插入和查找（m 为字符串长度）；用于自动补全、拼写检查、IP 路由最长前缀匹配；变种：压缩 Trie（Radix Tree）、后缀树（Suffix Tree）
- **同义词**：Prefix Tree（前缀树）、Digital Tree（数字树）
- **相关**：String（字符串）、Tree（树）、Autocomplete（自动补全）
- **例句**：A Trie can efficiently find all strings with a given prefix.
