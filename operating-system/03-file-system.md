# 03. 文件系统 (File System)

## 基础概念

### Absolute Path / 绝对路径
- **中文**：绝对路径
- **定义**：从根目录开始的完整路径
- **说明**：Unix 中绝对路径以 / 开头，如 /home/user/file.txt；Windows 中以盘符开头，如 C:\Users\user\file.txt
- **相关**：Relative Path（相对路径）、Root Directory（根目录）

### Access Control / 访问控制
- **中文**：访问控制
- **定义**：限制用户对系统资源访问的机制
- **说明**：访问控制包括自主访问控制（DAC）、强制访问控制（MAC）、基于角色的访问控制（RBAC）
- **相关**：Permission（权限）、ACL（访问控制列表）、Security（安全）

### Access Control List (ACL) / 访问控制列表
- **中文**：访问控制列表
- **定义**：为文件/目录指定额外访问权限的列表
- **说明**：ACL 扩展了传统 Unix 权限模型，允许为多个用户和组设置细粒度权限
- **相关**：Permission（权限）、Access Control（访问控制）

### Access Time (atime) / 访问时间
- **中文**：访问时间
- **定义**：文件最后一次被读取的时间
- **说明**：频繁更新访问时间会降低性能，现代系统通常使用 relatime 或 noatime 挂载选项减少更新
- **相关**：Timestamp（时间戳）、mtime、ctime

### Allocation Size / 分配大小
- **中文**：分配大小
- **定义**：文件系统在磁盘上为文件分配的实际空间大小
- **说明**：分配大小是块大小的整数倍；可能大于文件实际大小（内部碎片）
- **相关**：File Size（文件大小）、Block Size（块大小）

### Allocation Unit / 分配单元
- **中文**：分配单元
- **定义**：同 Cluster
- **相关**：Cluster（簇）

### APFS (Apple File System) / 苹果文件系统
- **中文**：苹果文件系统
- **定义**：Apple 开发的用于 macOS、iOS 等系统的文件系统
- **说明**：APFS 针对闪存优化，支持快照、克隆、空间共享、加密、写时复制等；替代 HFS+
- **相关**：HFS+、Snapshot（快照）、Copy-on-Write（写时复制）

### Backup / 备份
- **音标**：/ˈbækʌp/
- **中文**：备份
- **定义**：复制数据以防止数据丢失的操作
- **说明**：备份策略包括完全备份、增量备份、差异备份；备份可以存储在磁带、磁盘、云端等
- **相关**：Snapshot（快照）、Restore（恢复）、Archive（归档）

### Bad Block / 坏块
- **中文**：坏块
- **定义**：无法正常读写的闪存块或磁盘扇区
- **说明**：出厂时就有坏块（制造缺陷），使用过程中也会产生坏块；文件系统或 SSD 控制器管理坏块映射
- **相关**：SSD、HDD、Spare Area（预留空间）

### Bitmap / 位图
- **音标**：/ˈbɪtmæp/
- **中文**：位图
- **定义**：用每一位表示一个块或 inode 是否被占用的数据结构
- **说明**：块位图（Block Bitmap）跟踪数据块的使用情况；inode 位图（Inode Bitmap）跟踪 inode 的使用情况
- **相关**：Free List（空闲列表）、Block（块）、Inode（索引节点）

### Block / 块
- **音标**：/blɒk/
- **中文**：块
- **定义**：文件系统中数据存储的基本单位
- **说明**：块大小通常为 1KB、2KB、4KB；文件按块分配存储空间；块是文件系统和磁盘 I/O 的基本单位
- **相关**：Block Size（块大小）、Sector（扇区）、Cluster（簇）

### Block Bitmap / 块位图
- **中文**：块位图
- **定义**：跟踪数据块使用情况的位图
- **相关**：Bitmap（位图）、Inode Bitmap（索引节点位图）

### Block Group / 块组
- **中文**：块组
- **定义**：文件系统将磁盘空间划分为多个块组，每个块组包含超级块、块位图、inode 位图、inode 表和数据块
- **说明**：块组将文件系统的元数据和数据分散存储，提高性能和可靠性；ext2/3/4 使用块组
- **相关**：Superblock（超级块）、Bitmap（位图）、Inode（索引节点）

### Block Size / 块大小
- **中文**：块大小
- **定义**：文件系统中单个块的字节数
- **说明**：块大小影响文件系统的性能和空间利用率；大块减少碎片但增加内部碎片；小块节省空间但增加管理开销
- **相关**：Block（块）、Sector（扇区）

### Btrfs (B-tree File System) / B 树文件系统
- **发音**：/ˈbʌtər-fɪs/ 或 /ˈbiː-triː-fɪs/
- **中文**：B 树文件系统
- **定义**：Linux 的下一代文件系统，支持高级特性
- **说明**：Btrfs 支持写时复制（COW）、快照（Snapshot）、子卷（Subvolume）、校验和（Checksum）、RAID、压缩等
- **相关**：ZFS、ext4、Snapshot（快照）、Copy-on-Write（写时复制）

### Change Directory (cd) / 改变目录
- **中文**：改变目录
- **定义**：改变当前工作目录的命令/操作
- **说明**：cd 是 Shell 内置命令；chdir() 是系统调用
- **相关**：Working Directory（工作目录）、Path（路径）

### Change Time (ctime) / 状态改变时间
- **中文**：状态改变时间
- **定义**：文件元数据（权限、所有者等）最后一次被修改的时间
- **说明**：ctime 变化意味着 inode 信息被修改；注意不是创建时间（Creation Time）
- **相关**：Timestamp（时间戳）、atime、mtime

### Checksum / 校验和
- **音标**：/ˈtʃek-sʌm/
- **中文**：校验和
- **定义**：用于检测数据错误的计算值
- **说明**：文件系统使用校验和验证数据完整性；常见算法包括 CRC32、MD5、SHA-256
- **相关**：Data Integrity（数据完整性）、Error Detection（错误检测）

### chgrp
- **发音**：/siː-ɡruːp/ 或 /tʃeɪ-ɡruːp/
- **中文**：改变组（命令）
- **定义**：修改文件或目录所属组的命令
- **相关**：Group（组）、chown

## 虚拟文件系统

### chmod
- **发音**：/siː-mɒd/ 或 /tʃeɪ-mɒd/
- **中文**：改变权限（命令）
- **定义**：修改文件或目录权限的命令
- **说明**：chmod 可以使用符号模式（如 u+x, go-w）或八进制模式（如 755）；chmod() 是对应的系统调用
- **相关**：Permission（权限）、Mode（模式）

### chown
- **发音**：/siː-hoʊn/ 或 /tʃeɪ-hoʊn/
- **中文**：改变所有者（命令）
- **定义**：修改文件或目录所有者的命令
- **说明**：chown 可以同时修改所有者和组（如 chown user:group file）；chown() 是对应的系统调用
- **相关**：Owner（所有者）、Group（组）

### Close / 关闭
- **音标**：/kloʊz/
- **中文**：关闭
- **定义**：断开进程与文件的连接，释放文件描述符
- **说明**：close() 系统调用关闭文件；关闭后文件描述符可以被重用；未关闭文件可能导致资源泄漏
- **相关**：Open（打开）、File Descriptor（文件描述符）

### Cluster / 簇
- **音标**：/ˈklʌstər/
- **中文**：簇
- **定义**：文件系统分配空间的最小单位，由连续的扇区组成
- **说明**：Windows FAT/NTFS 使用簇的概念；簇大小通常为 4KB、8KB、16KB
- **同义词**：Allocation Unit
- **相关**：Block（块）、Sector（扇区）

### Copy / 复制
- **音标**：/ˈkɒpi/
- **中文**：复制
- **定义**：创建文件的副本
- **说明**：复制文件需要读取原文件数据并写入新文件；可以保留或修改元数据
- **相关**：Move（移动）、Create（创建）

### Create / 创建
- **音标**：/kriˈeɪt/
- **中文**：创建
- **定义**：创建新文件的操作
- **说明**：creat() 系统调用创建新文件（或截断已有文件）并打开；open() 配合 O_CREAT 标志也可以创建文件
- **相关**：Open（打开）、Delete（删除）

### Creation Time (crtime/btime) / 创建时间
- **中文**：创建时间
- **定义**：文件被创建的时间
- **说明**：Unix 传统文件系统（ext2/3）不记录创建时间；ext4、Btrfs、NTFS 等支持创建时间
- **相关**：Timestamp（时间戳）

### Current Directory / 当前目录
- **中文**：当前目录
- **定义**：同 Working Directory
- **相关**：Working Directory（工作目录）

### Current Working Directory (pwd) / 当前工作目录
- **中文**：当前工作目录
- **定义**：显示当前工作路径的命令
- **说明**：pwd 输出当前目录的绝对路径；getcwd() 是 C 库函数
- **相关**：Working Directory（工作目录）、Path（路径）

## 文件系统挂载与管理

### Cylinder / 柱面
- **音标**：/ˈsɪlɪndər/
- **中文**：柱面
- **定义**：磁盘上所有盘片同一半径的磁道组成的圆柱面
- **说明**：同一柱面的磁道不需要移动磁头即可访问；传统硬盘优化考虑柱面布局
- **相关**：Track（磁道）、Sector（扇区）、Head（磁头）

### Cylinder Group / 柱面组
- **中文**：柱面组
- **定义**：UFS/FFS 文件系统中，将同一柱面（Cylinder）的磁道组合在一起的单元
- **说明**：柱面组减少磁盘寻道时间，提高顺序读写性能；现代 SSD 不需要考虑柱面组
- **相关**：UFS、Cylinder（柱面）、Track（磁道）

### Dangling Link / 悬空链接
- **中文**：悬空链接
- **定义**：目标文件/目录已被删除的符号链接
- **说明**：访问悬空链接会报错；需要定期清理或更新符号链接
- **相关**：Symbolic Link（符号链接）

## 文件系统结构

### Data Block / 数据块
- **中文**：数据块
- **定义**：存储文件实际数据的块
- **说明**：inode 中包含指向数据块的指针；大文件需要间接块（Indirect Block）指向更多数据块
- **相关**：Inode（索引节点）、Indirect Block（间接块）

### Data Integrity / 数据完整性
- **中文**：数据完整性
- **定义**：确保数据在存储和传输过程中不被损坏或篡改的属性
- **说明**：文件系统通过校验和、日志、写时复制等技术保证数据完整性
- **相关**：Checksum（校验和）、Journaling（日志记录）、Copy-on-Write（写时复制）

## 文件系统性能与优化

### Defragmentation / 碎片整理
- **音标**：/diːˌfræɡmenˈteɪʃn/
- **中文**：碎片整理
- **定义**：重新组织磁盘上的文件，使每个文件的数据连续存储
- **说明**：碎片整理减少文件访问的寻道时间；传统 HDD 需要碎片整理，SSD 不需要（且会缩短寿命）
- **同义词**：Defrag
- **相关**：Fragmentation（碎片化）、Disk（磁盘）

### Delayed Allocation / 延迟分配
- **中文**：延迟分配
- **定义**：推迟实际的磁盘块分配，直到数据真正写入磁盘
- **说明**：延迟分配允许文件系统根据实际写入大小和数据模式选择最佳分配策略，减少碎片化
- **相关**：Allocation（分配）、ext4、XFS、Btrfs

### Delete / 删除
- **音标**：/dɪˈliːt/
- **中文**：删除
- **定义**：删除文件或目录的操作
- **说明**：unlink() 系统调用删除文件（减少链接计数）；rmdir() 删除空目录；rm 命令删除文件/目录
- **相关**：Create（创建）、Remove（移除）、unlink()

### dentry
- **发音**：/diː-ɛntri/
- **中文**：目录项缓存
- **定义**：VFS 中缓存目录项（文件名到 inode 映射）的数据结构
- **说明**：dentry 缓存加速路径解析；目录项缓存（dentry cache/dcache）是 Linux 内核的重要缓存
- **相关**：VFS（虚拟文件系统）、Inode（索引节点）、Path（路径）

### devfs / dev 文件系统
- **发音**：/dev-ef-es/
- **中文**：dev 文件系统
- **定义**：管理设备文件的虚拟文件系统
- **说明**：现代 Linux 使用 udev（用户空间设备管理）动态创建 /dev 下的设备节点；早期使用 devfs
- **相关**：udev、Device File（设备文件）

## 磁盘与存储

### Direct Block / 直接块
- **中文**：直接块
- **定义**：inode 中直接指向的数据块
- **说明**：inode 中通常有多个直接块指针（如 ext2 有 12 个）；小文件可以直接通过直接块访问
- **相关**：Indirect Block（间接块）、Inode（索引节点）

### Directory / 目录
- **音标**：/dəˈrektəri/
- **中文**：目录（文件夹）
- **定义**：用于组织和管理文件的特殊文件，包含文件名到文件位置的映射
- **说明**：目录可以嵌套形成树形结构；根目录（Root Directory）是文件系统的起点
- **同义词**：Folder（文件夹）
- **相关**：File（文件）、Path（路径）、Root Directory（根目录）

### Disk / 磁盘
- **音标**：/dɪsk/
- **中文**：磁盘
- **定义**：用于持久存储数据的磁性或固态存储设备
- **说明**：包括硬盘驱动器（HDD，使用磁性盘片）和固态驱动器（SSD，使用闪存）
- **相关**：HDD、SSD、Storage（存储）

### Duplicate / 复制（dup）
- **中文**：复制
- **定义**：复制文件描述符的系统调用
- **说明**：dup() 复制文件描述符到最小的未使用描述符；dup2() 复制到指定描述符；复制后两个描述符共享文件偏移量
- **相关**：File Descriptor（文件描述符）、Redirection（重定向）

### Execute (x) / 执行
- **中文**：执行
- **定义**：执行文件（程序/脚本）或进入目录的权限
- **相关**：Permission（权限）、Read（读）、Write（写）

### exFAT (Extended File Allocation Table) / 扩展文件分配表
- **中文**：扩展文件分配表
- **定义**：微软设计的适用于闪存的文件系统
- **说明**：exFAT 支持大文件（16EB）和大分区（128PB），无日志功能， overhead 低；常用于 U 盘和 SD 卡
- **相关**：FAT32、NTFS

### ext2 (Second Extended File System) / 第二扩展文件系统
- **中文**：第二扩展文件系统
- **定义**：Linux 早期的文件系统，无日志功能
- **说明**：ext2 结构简单、性能高；但不支持日志，崩溃后需要完整文件系统检查（fsck）
- **相关**：ext3、ext4、Linux

### ext3 (Third Extended File System) / 第三扩展文件系统
- **中文**：第三扩展文件系统
- **定义**：在 ext2 基础上增加日志功能的 Linux 文件系统
- **说明**：ext3 兼容 ext2，可以在线升级；支持三种日志模式（Journal、Ordered、Writeback）
- **相关**：ext2、ext4、Journaling（日志记录）

### ext4 (Fourth Extended File System) / 第四扩展文件系统
- **中文**：第四扩展文件系统
- **定义**：Linux 的默认文件系统，ext3 的后续版本
- **说明**：ext4 支持更大的文件（16TB）和文件系统（1EB）、Extents（连续块分配）、延迟分配、多块分配器、快速文件系统检查等
- **相关**：ext3、Extents、Linux

### Extension / 扩展名
- **音标**：/ɪkˈstenʃn/
- **中文**：扩展名
- **定义**：文件名中用于标识文件类型的后缀
- **说明**：如 .txt（文本）、.exe（可执行文件）、.jpg（图片）；Windows 使用扩展名关联程序，Unix 使用文件头（Magic Number）
- **相关**：File Name（文件名）、File Type（文件类型）

### Extent / 区段
- **音标**：/ɪkˈstent/
- **中文**：区段
- **定义**：文件系统中连续的块范围，作为一个分配单元
- **说明**：Extents 减少大文件的碎片化和管理开销；ext4、XFS 使用 extents 替代传统块指针
- **相关**：Block（块）、Contiguous Allocation（连续分配）

### fallocate
- **发音**：/ef-ˈæləkeɪt/
- **中文**：文件预分配（系统调用）
- **定义**：预先为文件分配磁盘空间的系统调用
- **说明**：fallocate 比写入零字节更快，不实际写入数据，只分配空间；支持 FALLOC_FL_PUNCH_HOLE 释放空间
- **相关**：Preallocation（预分配）、Truncate（截断）

## 权限与访问控制

### FAT32 (File Allocation Table 32) / 32 位文件分配表
- **中文**：32 位文件分配表
- **定义**：FAT 文件系统的 32 位版本
- **说明**：FAT32 兼容性好，被几乎所有操作系统支持；但单文件最大 4GB，分区最大 2TB，无日志功能
- **相关**：FAT16、exFAT、NTFS

### fdatasync
- **发音**：/ef-deɪtə-sɪŋk/
- **中文**：数据同步（系统调用）
- **定义**：将指定文件的数据强制写入磁盘，但不同步元数据（除非元数据影响数据读取）
- **说明**：fdatasync 比 fsync 快，因为不需要同步所有元数据
- **相关**：fsync、Sync（同步）

## 目录操作

### File / 文件
- **音标**：/faɪl/
- **中文**：文件
- **定义**：存储在存储设备上的命名数据集合
- **说明**：文件是操作系统管理数据的基本单位；包含数据和元数据（文件名、大小、创建时间、权限等）
- **相关**：Directory（目录）、File System（文件系统）、Inode（索引节点）

### FILE
- **发音**：/faɪl/
- **中文**：文件结构（C 标准库）
- **定义**：C 标准库中定义的文件流结构体
- **说明**：FILE 结构包含文件描述符、缓冲区、缓冲状态、错误标志等；stdio 函数操作 FILE 而非直接使用文件描述符
- **相关**：File Pointer（文件指针）、fopen、fclose

### File Allocation Table (FAT) / 文件分配表
- **中文**：文件分配表
- **定义**：FAT 文件系统中用于记录文件块链接关系的表
- **说明**：FAT 表中每个条目对应一个块，存储下一个块的编号或结束标记；文件通过 FAT 表链式存储
- **相关**：FAT12、FAT16、FAT32、Cluster（簇）

### File Attribute / 文件属性
- **中文**：文件属性
- **定义**：文件的元数据信息
- **说明**：包括文件类型、大小、时间戳、权限、所有者等
- **相关**：Metadata（元数据）、Inode（索引节点）

### File Description / 文件描述
- **中文**：文件描述
- **定义**：内核中描述已打开文件状态的数据结构（与文件描述符区分）
- **说明**：文件描述包含文件偏移量、打开模式、指向 inode 或 vnode 的指针等；多个文件描述符可以指向同一文件描述（如 dup 后）
- **相关**：File Descriptor（文件描述符）、Inode（索引节点）

### File Descriptor / 文件描述符
- **中文**：文件描述符
- **定义**：操作系统分配给已打开文件的非负整数标识符
- **说明**：标准输入（stdin）为 0，标准输出（stdout）为 1，标准错误（stderr）为 2；文件描述符是进程级资源
- **相关**：Open（打开）、Close（关闭）、File Description（文件描述）

### File Name / 文件名
- **中文**：文件名
- **定义**：标识文件的字符串
- **说明**：文件名在同一目录下必须唯一；不同文件系统对文件名长度和字符有不同限制
- **相关**：Extension（扩展名）、Path（路径）

### File Offset / 文件偏移量
- **中文**：文件偏移量
- **定义**：文件当前读写位置相对于文件开头的字节数
- **说明**：每次读写后偏移量自动前进；可以通过 seek 改变偏移量
- **相关**：Seek（定位）、Read（读取）、Write（写入）

### File Pointer / 文件指针
- **中文**：文件指针
- **定义**：C 标准库中 FILE* 类型，封装了文件描述符和缓冲信息
- **说明**：fopen() 返回文件指针，内部使用文件描述符；fread/fwrite 使用缓冲减少系统调用次数
- **相关**：FILE、File Descriptor（文件描述符）

### File Size / 文件大小
- **中文**：文件大小
- **定义**：文件包含的数据字节数
- **说明**：文件大小可能小于分配大小（Allocation Size），因为文件系统按块分配空间
- **相关**：Allocation Size（分配大小）、Block Size（块大小）

### File System / 文件系统
- **中文**：文件系统
- **定义**：操作系统中负责管理和组织文件存储、检索、命名的子系统
- **说明**：文件系统提供文件的逻辑视图，屏蔽物理存储细节；包括文件组织、存储空间管理、目录管理、文件保护等功能
- **相关**：File（文件）、Directory（目录）、Volume（卷）、Partition（分区）

### File System Type / 文件系统类型
- **中文**：文件系统类型
- **定义**：文件系统的具体实现格式
- **说明**：不同文件系统类型有不同的特性和适用场景
- **相关**：ext4、NTFS、FAT32、XFS、Btrfs

### File Type / 文件类型
- **中文**：文件类型
- **定义**：文件的分类，如普通文件、目录、设备文件等
- **说明**：Unix 中文件类型包括：普通文件（-）、目录（d）、字符设备（c）、块设备（b）、符号链接（l）、管道（p）、套接字（s）
- **相关**：File（文件）、Extension（扩展名）、Magic Number（魔数）

### Filesystem in Userspace (FUSE) / 用户空间文件系统
- **发音**：/fjuːz/
- **中文**：用户空间文件系统
- **定义**：允许在用户空间实现文件系统的框架
- **说明**：FUSE 通过内核模块将文件系统请求转发到用户空间程序处理；用于实现 SSHFS、NTFS-3G 等
- **相关**：File System（文件系统）、VFS（虚拟文件系统）

### Flash Memory / 闪存
- **中文**：闪存
- **定义**：一种非易失性存储器，可以电擦写
- **说明**：闪存包括 NOR Flash（随机读取快，用于固件）和 NAND Flash（密度高、成本低，用于 SSD/U 盘）
- **相关**：SSD、EEPROM、NAND

### Flush / 刷新
- **音标**：/flʌʃ/
- **中文**：刷新
- **定义**：强制将缓冲区中的数据写入磁盘
- **说明**：fflush() 刷新文件流缓冲区；fsync() 将文件数据强制写入磁盘；sync() 将所有缓冲区数据写入磁盘
- **相关**：Buffer（缓冲区）、Sync（同步）、fsync()

### Folder / 文件夹
- **音标**：/ˈfoʊldər/
- **中文**：文件夹
- **定义**：同 Directory，图形界面中的常用术语
- **相关**：Directory（目录）

### Fragmentation / 碎片化
- **中文**：碎片化
- **定义**：文件数据分散存储在磁盘不连续位置的状态
- **说明**：碎片化降低文件访问性能；文件系统通过预分配、延迟分配等技术减少碎片化
- **相关**：Defragmentation（碎片整理）、Allocation（分配）

### Free List / 空闲列表
- **中文**：空闲列表
- **定义**：记录空闲块的链表或数据结构
- **说明**：文件系统通过空闲列表管理可用空间；分配块时从空闲列表中移除，释放时加入
- **相关**：Bitmap（位图）、Block（块）

### fsync
- **发音**：/ef-sɪŋk/
- **中文**：文件同步（系统调用）
- **定义**：将指定文件的所有数据和元数据强制写入磁盘的系统调用
- **说明**：fsync 确保数据持久化，常用于事务型应用；但会降低性能
- **相关**：Sync（同步）、fdatasync、Flush（刷新）

### Garbage Collection (SSD) / 垃圾回收（SSD）
- **中文**：垃圾回收（SSD）
- **定义**：SSD 控制器回收无效数据块的过程
- **说明**：SSD 不能直接覆盖写入，需要先擦除块；垃圾回收将有效数据移到新块，然后擦除旧块
- **相关**：SSD、TRIM、Wear Leveling（磨损均衡）

### Group / 组
- **音标**：/ɡruːp/
- **中文**：组
- **定义**：用户的集合，用于批量管理权限
- **说明**：文件的组权限适用于该组的所有成员；chgrp 命令改变文件所属组
- **相关**：Owner（所有者）、Permission（权限）

### GUID Partition Table (GPT) / GUID 分区表
- **中文**：GUID 分区表
- **定义**：UEFI 系统中使用的分区表格式
- **说明**：GPT 使用 GUID 标识分区，支持最多 128 个分区，最大支持 18EB 磁盘；有备份分区表提高可靠性
- **相关**：MBR、UEFI、Partition Table（分区表）

### Hard Disk Drive (HDD) / 硬盘驱动器
- **中文**：硬盘驱动器
- **定义**：使用旋转磁性盘片存储数据的机械设备
- **说明**：HDD 容量大、价格低，但速度较慢、有机械故障风险；包含盘片、磁头、马达等机械部件
- **相关**：SSD、Disk（磁盘）、Platter（盘片）

### Hard Link / 硬链接
- **中文**：硬链接
- **定义**：指向文件 inode 的目录项，与原始文件共享相同的物理数据
- **说明**：硬链接与原始文件完全等价，删除其中一个不影响另一个；不能跨文件系统，不能链接目录
- **相关**：Symbolic Link（符号链接）、Inode（索引节点）、Link Count（链接计数）

### Head / 磁头
- **音标**：/hed/
- **中文**：磁头
- **定义**：磁盘驱动器中用于读写数据的部件
- **说明**：每个盘片有上下两个磁头；磁头移动到指定磁道进行读写
- **相关**：Track（磁道）、Sector（扇区）

### HFS+ (Hierarchical File System Plus) / 分层文件系统增强版
- **中文**：分层文件系统增强版
- **定义**：Apple 早期的 macOS 文件系统
- **说明**：HFS+ 已被 APFS 取代；支持日志、压缩、磁盘配额等
- **相关**：APFS、HFS

### Indexed Allocation / 索引分配
- **中文**：索引分配
- **定义**：每个文件有一个索引块，存储该文件所有数据块的指针
- **说明**：索引分配支持直接访问，无外部碎片；Unix inode 使用索引分配（直接块+间接块）
- **相关**：Contiguous Allocation（连续分配）、Linked Allocation（链接分配）、Inode（索引节点）

### Indirect Block / 间接块
- **中文**：间接块
- **定义**：不存储文件数据，而是存储指向其他数据块指针的块
- **说明**：分为一级间接块（Single Indirect）、二级间接块（Double Indirect）、三级间接块（Triple Indirect）；用于支持大文件
- **相关**：Inode（索引节点）、Data Block（数据块）

### Inode / 索引节点
- **音标**：/ˈaɪnoʊd/
- **中文**：索引节点
- **定义**：Unix 文件系统中存储文件元数据的数据结构
- **说明**：每个文件对应一个 inode；inode 包含文件类型、权限、所有者、时间戳、大小、数据块指针等；文件名存储在目录中
- **相关**：File（文件）、Directory（目录）、Data Block（数据块）

### Inode Bitmap / 索引节点位图
- **中文**：索引节点位图
- **定义**：跟踪 inode 使用情况的位图
- **相关**：Bitmap（位图）、Block Bitmap（块位图）

### Journal / 日志
- **音标**：/ˈdʒɜːrnl/
- **中文**：日志
- **定义**：记录文件系统操作的日志，用于故障恢复
- **说明**：日志文件系统（Journaling File System）在修改元数据前先写日志；系统崩溃后可以通过日志恢复一致性
- **相关**：Journaling（日志记录）、Transaction（事务）、Recovery（恢复）

### Journaling / 日志记录
- **中文**：日志记录
- **定义**：将文件系统操作记录到日志中的技术
- **说明**：日志记录模式包括：写日志（Writeback，只记录元数据）、有序（Ordered，数据先于元数据写入）、数据（Data，记录数据和元数据）
- **相关**：Journal（日志）、File System（文件系统）、ext3、ext4

### journaling File System / 日志文件系统
- **中文**：日志文件系统
- **定义**：使用日志记录机制保证一致性的文件系统
- **说明**：如 ext3、ext4、XFS、NTFS、HFS+；崩溃后不需要完整检查文件系统
- **相关**：Journal（日志）、ext3、ext4、XFS

## 文件系统类型

### Link Count / 链接计数
- **中文**：链接计数
- **定义**：指向某个 inode 的硬链接数量
- **说明**：链接计数为 0 时，文件数据被删除；创建硬链接时链接计数加 1，删除时减 1
- **相关**：Hard Link（硬链接）、Inode（索引节点）

### Linked Allocation / 链接分配
- **定义**：将文件数据存储在不连续的块中，每个块包含指向下一个块的指针
- **说明**：链接分配无外部碎片，但随机访问慢，指针占用空间；FAT 是链接分配的变体
- **相关**：Contiguous Allocation（连续分配）、Indexed Allocation（索引分配）、FAT

### List Directory (ls) / 列出目录
- **中文**：列出目录
- **定义**：列出目录内容的命令
- **说明**：ls 命令列出文件名；ls -l 显示详细信息；ls -a 显示隐藏文件
- **相关**：Directory（目录）、File（文件）

### Logical Volume / 逻辑卷
- **中文**：逻辑卷
- **定义**：逻辑卷管理（LVM）中的虚拟存储单元
- **说明**：逻辑卷可以跨越多个物理磁盘，可以动态调整大小；LVM 提供物理卷（PV）、卷组（VG）、逻辑卷（LV）三层抽象
- **相关**：LVM、Physical Volume（物理卷）、Volume Group（卷组）

### Logical Volume Manager (LVM) / 逻辑卷管理器
- **中文**：逻辑卷管理器
- **定义**：提供逻辑存储抽象，允许动态管理磁盘空间的系统
- **说明**：LVM 将物理磁盘抽象为物理卷（PV），组合成卷组（VG），再划分为逻辑卷（LV）；支持快照、条带化、镜像等
- **相关**：Logical Volume（逻辑卷）、Physical Volume（物理卷）、Volume Group（卷组）

### Magic Number / 魔数
- **中文**：魔数
- **定义**：文件开头的特殊字节序列，用于标识文件类型
- **说明**：如 PDF 文件以 %PDF 开头，JPEG 以 FF D8 FF 开头；Unix file 命令使用魔数识别文件类型
- **相关**：File Type（文件类型）、Header（文件头）

### Make Directory (mkdir) / 创建目录
- **中文**：创建目录
- **定义**：创建新目录的系统调用/命令
- **说明**：mkdir() 系统调用创建目录；mkdir -p 可以递归创建父目录
- **相关**：Remove Directory（删除目录）、Directory（目录）

### Master Boot Record (MBR) / 主引导记录
- **中文**：主引导记录
- **定义**：传统 BIOS 系统中磁盘起始扇区的数据结构，包含引导代码和分区表
- **说明**：MBR 支持最多 4 个主分区或 3 个主分区+1 个扩展分区；最大支持 2TB 磁盘
- **相关**：GPT、BIOS、Partition Table（分区表）

### Master File Table (MFT) / 主文件表
- **中文**：主文件表
- **定义**：NTFS 文件系统中存储所有文件和目录元数据的特殊文件
- **说明**：MFT 由多个记录组成，每个记录 1KB；小文件的数据直接存储在 MFT 记录中（常驻属性），大文件的数据存储在独立块中（非常驻属性）
- **相关**：NTFS、Inode（索引节点）、Attribute（属性）

### Metadata / 元数据
- **音标**：/ˈmetədeɪtə/
- **中文**：元数据
- **定义**：描述文件属性和管理信息的数据
- **说明**：元数据包括文件名、大小、创建时间、修改时间、访问时间、权限、所有者等
- **相关**：Inode（索引节点）、File Attribute（文件属性）

### Mirror / 镜像
- **音标**：/ˈmɪrər/
- **中文**：镜像
- **定义**：将数据完整复制到另一个磁盘的技术
- **说明**：镜像提供冗余，一个磁盘故障时数据不丢失；RAID 1 使用镜像
- **相关**：RAID、Redundancy（冗余）

### MLC (Multi-Level Cell) / 多层单元
- **中文**：多层单元
- **定义**：每个存储单元存储 2 位数据的闪存
- **说明**：MLC 在速度、寿命、成本之间取得平衡；曾广泛用于消费级 SSD
- **相关**：SLC、TLC、QLC、NAND

### Mode / 模式
- **音标**：/moʊd/
- **中文**：模式
- **定义**：Unix 中文件权限和类型的组合
- **说明**：模式通常用八进制表示，如 755（rwxr-xr-x）、644（rw-r--r--）；包含文件类型和权限位
- **相关**：Permission（权限）、umask

### Modification Time (mtime) / 修改时间
- **中文**：修改时间
- **定义**：文件内容最后一次被修改的时间
- **说明**：mtime 变化意味着文件数据被修改
- **相关**：Timestamp（时间戳）、atime、ctime

### Mount / 挂载
- **音标**：/maʊnt/
- **中文**：挂载
- **定义**：将文件系统附加到目录树中的操作
- **说明**：mount() 系统调用将设备上的文件系统挂载到指定挂载点；卸载使用 umount()
- **相关**：Unmount（卸载）、Mount Point（挂载点）、File System（文件系统）

### Mount Point / 挂载点
- **中文**：挂载点
- **定义**：文件系统中用于挂载其他文件系统的目录
- **说明**：挂载后，访问挂载点目录就是访问被挂载的文件系统根目录；如 /mnt、/media
- **相关**：Mount（挂载）、Root Directory（根目录）

### Move / 移动
- **音标**：/muːv/
- **中文**：移动
- **定义**：将文件从一个位置移动到另一个位置
- **说明**：同一文件系统内移动通常是重命名 inode 链接；跨文件系统移动需要复制后删除原文件
- **相关**：Rename（重命名）、Copy（复制）、Delete（删除）

### NAND / NAND 闪存
- **发音**：/nænd/
- **中文**：NAND 闪存
- **定义**：一种闪存技术，使用 NAND 逻辑门结构
- **说明**：NAND 闪存密度高、成本低，是 SSD 和 U 盘的主要存储介质；分为 SLC、MLC、TLC、QLC
- **相关**：Flash Memory（闪存）、SSD、SLC、MLC、TLC、QLC

### NTFS (New Technology File System) / 新技术文件系统
- **中文**：新技术文件系统
- **定义**：Windows 的默认文件系统
- **说明**：NTFS 支持大文件、权限控制、加密、压缩、磁盘配额、硬链接、符号链接、日志记录等
- **相关**：FAT32、Windows、MFT（主文件表）

### Open / 打开
- **音标**：/ˈoʊpən/
- **中文**：打开
- **定义**：建立进程与文件之间的连接，返回文件描述符
- **说明**：open() 系统调用打开文件，操作系统检查权限、定位文件、分配文件描述符；打开后可以进行读写操作
- **相关**：Close（关闭）、File Descriptor（文件描述符）、open()（系统调用）

### Over-provisioning / 过度配置
- **中文**：过度配置
- **定义**：为 SSD 预留额外空间以提高性能和寿命的技术
- **说明**：用户可以分配小于 SSD 总容量的分区来增加预留空间；企业级 SSD 通常有更高的过度配置比例
- **相关**：SSD、Spare Area（预留空间）

### Owner / 所有者
- **音标**：/ˈoʊnər/
- **中文**：所有者
- **定义**：文件的拥有用户
- **说明**：文件的所有者可以改变文件权限；chown 命令改变所有者
- **相关**：Group（组）、Permission（权限）

### Parent Directory / 父目录
- **中文**：父目录
- **定义**：包含当前目录的上一级目录
- **说明**：在路径中用 .. 表示；根目录的父目录是其自身
- **相关**：Directory（目录）、Path（路径）

### Partition / 分区
- **音标**：/pɑːrˈtɪʃn/
- **中文**：分区
- **定义**：将物理磁盘划分为多个独立区域的操作
- **说明**：分区表记录分区信息；MBR（主引导记录）和 GPT（GUID 分区表）是两种分区方案
- **相关**：Volume（卷）、MBR、GPT

### Partition Table / 分区表
- **中文**：分区表
- **定义**：记录磁盘分区信息的数据结构
- **相关**：Partition（分区）、MBR、GPT

### Path / 路径
- **音标**：/pæθ/
- **中文**：路径
- **定义**：标识文件或目录在文件系统中位置的字符串
- **说明**：绝对路径（Absolute Path）从根目录开始；相对路径（Relative Path）从当前目录开始
- **相关**：Absolute Path（绝对路径）、Relative Path（相对路径）、Working Directory（工作目录）

### Permission / 权限
- **音标**：/pərˈmɪʃn/
- **中文**：权限
- **定义**：控制用户对文件/目录访问操作的规则
- **说明**：Unix 权限包括读（Read）、写（Write）、执行（Execute），分别对应所有者（Owner）、组（Group）、其他（Other）
- **相关**：Access Control（访问控制）、ACL（访问控制列表）、Mode（模式）

### Physical Volume (PV) / 物理卷
- **中文**：物理卷
- **定义**：LVM 中最底层的存储单元，对应物理磁盘或分区
- **相关**：LVM、Volume Group（卷组）

### Platter / 盘片
- **音标**：/ˈplætər/
- **中文**：盘片
- **定义**：硬盘中涂有磁性材料的圆形盘片，用于存储数据
- **说明**：一个硬盘可以有多个盘片，每个盘片有上下两个磁头
- **相关**：HDD、Track（磁道）、Sector（扇区）

### Preallocation / 预分配
- **中文**：预分配
- **定义**：在写入数据之前预先分配磁盘空间
- **说明**：预分配保证后续写入不会失败（空间不足），减少碎片化；常用于数据库和日志文件
- **相关**：Allocation（分配）、fallocate()

### procfs / proc 文件系统
- **发音**：/prɒk-ef-es/
- **中文**：proc 文件系统
- **定义**：Unix/Linux 中的虚拟文件系统，将内核数据结构暴露为文件
- **说明**：/proc 目录包含进程信息（/proc/PID）、系统信息（/proc/cpuinfo、/proc/meminfo）等；不是真实磁盘文件系统
- **相关**：sysfs、Virtual File System（虚拟文件系统）

### QLC (Quad-Level Cell) / 四层单元
- **中文**：四层单元
- **定义**：每个存储单元存储 4 位数据的闪存
- **说明**：QLC 成本最低、容量最大，但速度和寿命最低；适合大容量存储和读取密集型应用
- **相关**：SLC、MLC、TLC、NAND

### RAID (Redundant Array of Independent Disks) / 独立磁盘冗余阵列
- **发音**：/reɪd/
- **中文**：独立磁盘冗余阵列
- **定义**：将多个物理磁盘组合成一个逻辑单元，以提高性能或可靠性的技术
- **说明**：RAID 级别包括：RAID 0（条带化，性能高无冗余）、RAID 1（镜像，冗余高）、RAID 5（分布式奇偶校验）、RAID 6（双奇偶校验）、RAID 10（镜像+条带化）
- **相关**：Disk（磁盘）、Redundancy（冗余）、Stripe（条带化）、Mirror（镜像）

### ramfs / ram 文件系统
- **发音**：/ræm-ef-es/
- **中文**：ram 文件系统
- **定义**：完全基于内存的文件系统，不使用交换空间
- **说明**：ramfs 可以无限增长直到内存耗尽；tmpfs 是其改进版，可以使用交换空间
- **相关**：tmpfs、Virtual File System（虚拟文件系统）

### Read / 读取
- **音标**：/riːd/
- **中文**：读取
- **定义**：从文件中读取数据到内存
- **说明**：read() 系统调用从当前文件偏移量处读取指定字节数的数据；读取后偏移量前进
- **相关**：Write（写入）、File Offset（文件偏移量）

### Read (r) / 读
- **中文**：读
- **定义**：查看文件内容或列出目录内容的权限
- **相关**：Permission（权限）、Write（写）、Execute（执行）

### Redirection / 重定向
- **音标**：/rɪˌdɪrekˈʃn/
- **中文**：重定向
- **定义**：改变标准输入/输出/错误的来源或目标
- **说明**：Shell 中 > 重定向 stdout，< 重定向 stdin，2> 重定向 stderr；>& 合并输出流
- **相关**：Standard Input（标准输入）、Standard Output（标准输出）、Pipe（管道）

### ReiserFS
- **发音**：/ˈraɪzər-ef-es/
- **中文**：ReiserFS 文件系统
- **定义**：Linux 的日志文件系统，以高效处理小文件著称
- **说明**：ReiserFS 使用 B+ 树组织数据，适合大量小文件的场景；Reiser4 是后续版本
- **相关**：ext3、Journaling（日志记录）

### Relative Path / 相对路径
- **中文**：相对路径
- **定义**：相对于当前工作目录的路径
- **说明**：不以 / 或盘符开头；使用 .（当前目录）和 ..（父目录）表示相对位置
- **相关**：Absolute Path（绝对路径）、Working Directory（工作目录）

### Remove / 移除
- **音标**：/rɪˈmuːv/
- **中文**：移除
- **定义**：删除文件或目录的通用操作
- **说明**：remove() C 标准库函数可以删除文件或空目录；内部调用 unlink 或 rmdir
- **相关**：Delete（删除）、unlink()、rmdir()

### Remove Directory (rmdir) / 删除目录
- **中文**：删除目录
- **定义**：删除空目录的系统调用/命令
- **说明**：rmdir() 只能删除空目录；rm -r 可以递归删除目录及其内容
- **相关**：Make Directory（创建目录）、Delete（删除）

### Rename / 重命名
- **音标**：/ˌriːˈneɪm/
- **中文**：重命名
- **定义**：改变文件或目录的名称
- **说明**：rename() 系统调用重命名文件；在同一文件系统内是原子操作；可以移动文件到不同目录
- **相关**：File Name（文件名）、Move（移动）

### Restore / 恢复
- **音标**：/rɪˈstɔːr/
- **中文**：恢复
- **定义**：从备份中还原数据的操作
- **相关**：Backup（备份）、Recovery（恢复）

### Root Directory / 根目录
- **中文**：根目录
- **定义**：文件系统的最顶层目录
- **说明**：Unix 中根目录为 /；Windows 中每个分区有自己的根目录（如 C:\）
- **相关**：Path（路径）、Directory（目录）

### Rotational Latency / 旋转延迟
- **中文**：旋转延迟
- **定义**：盘片旋转到目标扇区到达磁头下方所需的时间
- **说明**：旋转延迟平均为半圈时间；7200 RPM 硬盘的平均旋转延迟约 4.17ms
- **相关**：Seek Time（寻道时间）、Transfer Time（传输时间）

### Sector / 扇区
- **音标**：/ˈsektər/
- **中文**：扇区
- **定义**：磁盘上数据存储的物理单位
- **说明**：传统硬盘扇区大小为 512 字节或 4KB（高级格式化）；文件系统块由多个扇区组成
- **相关**：Block（块）、Disk（磁盘）、Track（磁道）

### Seek / 定位
- **音标**：/siːk/
- **中文**：定位
- **定义**：改变文件的当前读写位置（偏移量）
- **说明**：lseek() 系统调用可以绝对定位（SEEK_SET）、相对定位（SEEK_CUR）、尾部定位（SEEK_END）
- **相关**：File Offset（文件偏移量）、Read（读取）、Write（写入）

### Seek Time / 寻道时间
- **中文**：寻道时间
- **定义**：磁头移动到目标磁道所需的时间
- **说明**：寻道时间是磁盘 I/O 的主要开销之一；传统硬盘寻道时间约 3-10ms
- **相关**：Rotational Latency（旋转延迟）、Transfer Time（传输时间）

### Setgid / 设置组 ID
- **发音**：/set-gee-eye-dee/
- **中文**：设置组 ID
- **定义**：文件权限位，允许用户以文件所属组的权限执行该文件，或在目录中创建的文件继承目录的组
- **相关**：Setuid（设置用户 ID）、Permission（权限）

### Setuid / 设置用户 ID
- **发音**：/set-you-eye-dee/
- **中文**：设置用户 ID
- **定义**：文件权限位，允许用户以文件所有者的权限执行该文件
- **说明**：Setuid 位用 s 表示（如 rwsr-xr-x）；常用于 passwd、ping 等需要特权的程序；安全风险较大
- **相关**：Setgid（设置组 ID）、Sticky Bit（粘滞位）、Permission（权限）

### SLC (Single-Level Cell) / 单层单元
- **中文**：单层单元
- **定义**：每个存储单元存储 1 位数据的闪存
- **说明**：SLC 速度快、寿命长、可靠性高，但成本高、容量小；用于企业级 SSD 和缓存
- **相关**：MLC、TLC、QLC、NAND

### Snapshot / 快照
- **音标**：/ˈsnæpʃɒt/
- **中文**：快照
- **定义**：文件系统或卷在某一时刻的只读副本
- **说明**：快照使用写时复制（COW）技术，初始不占用额外空间；常用于备份和数据保护
- **相关**：Copy-on-Write（写时复制）、Backup（备份）、LVM、Btrfs、ZFS

### Soft Link / 软链接
- **中文**：软链接
- **定义**：同 Symbolic Link
- **相关**：Symbolic Link（符号链接）

### Solid State Drive (SSD) / 固态驱动器
- **中文**：固态驱动器
- **定义**：使用闪存（Flash Memory）存储数据的电子设备
- **说明**：SSD 速度快、无噪音、抗震，但价格较高、有写入次数限制；现代系统首选存储设备
- **相关**：HDD、Flash Memory（闪存）、NAND

### Spare Area / 预留空间
- **中文**：预留空间
- **定义**：SSD 中保留的一部分存储空间，不暴露给用户
- **说明**：预留空间用于磨损均衡、坏块替换、垃圾回收；通常占总容量的 7-28%
- **相关**：SSD、Over-provisioning（过度配置）

### Standard Error (stderr) / 标准错误
- **中文**：标准错误
- **定义**：进程默认的错误输出流，通常对应屏幕
- **说明**：文件描述符为 2；与 stdout 分离，即使 stdout 被重定向，错误信息仍可以显示
- **相关**：Standard Input（标准输入）、Standard Output（标准输出）

### Standard Input (stdin) / 标准输入
- **中文**：标准输入
- **定义**：进程默认的输入流，通常对应键盘
- **说明**：文件描述符为 0；可以通过重定向（<）从文件读取
- **相关**：Standard Output（标准输出）、Standard Error（标准错误）、Redirection（重定向）

### Standard Output (stdout) / 标准输出
- **中文**：标准输出
- **定义**：进程默认的输出流，通常对应屏幕
- **说明**：文件描述符为 1；可以通过重定向（>）输出到文件
- **相关**：Standard Input（标准输入）、Standard Error（标准错误）、Redirection（重定向）

### Sticky Bit / 粘滞位
- **中文**：粘滞位
- **定义**：目录权限位，限制只有文件所有者才能删除该目录中的文件
- **说明**：粘滞位用 t 表示（如 rwxrwxrwt）；常用于 /tmp 等公共目录，防止用户删除他人的临时文件
- **相关**：Permission（权限）、/tmp

### Stripe / 条带化
- **音标**：/straɪp/
- **中文**：条带化
- **定义**：将数据分块分散存储到多个磁盘的技术
- **说明**：条带化提高读写性能；RAID 0 使用纯条带化，RAID 5/6 结合条带化和奇偶校验
- **相关**：RAID、Mirror（镜像）

### Subdirectory / 子目录
- **中文**：子目录
- **定义**：位于当前目录下的目录
- **相关**：Directory（目录）、Parent Directory（父目录）

## 文件属性与元数据

### Superblock / 超级块
- **音标**：/ˈsuːpərblɒk/
- **中文**：超级块
- **定义**：文件系统中存储文件系统整体元数据的结构
- **说明**：超级块包含块大小、块总数、空闲块数、inode 总数、空闲 inode 数、挂载时间等；是文件系统的关键结构
- **相关**：File System（文件系统）、Block（块）、Inode（索引节点）

### Symbolic Link / 符号链接
- **中文**：符号链接（软链接）
- **定义**：包含目标文件/目录路径的特殊文件
- **说明**：符号链接可以跨文件系统，可以链接目录；删除目标文件后符号链接成为悬空链接（Dangling Link）
- **同义词**：Soft Link、Symlink
- **相关**：Hard Link（硬链接）、Dangling Link（悬空链接）

### Symlink / 符号链接
- **中文**：符号链接
- **定义**：Symbolic Link 的缩写
- **相关**：Symbolic Link（符号链接）

### Sync / 同步
- **音标**：/sɪŋk/
- **中文**：同步
- **定义**：将内存中的文件数据同步到磁盘
- **说明**：sync() 系统调用将所有修改过的缓冲区写入磁盘；fsync() 同步单个文件；fdatasync() 只同步数据不同步元数据
- **相关**：Flush（刷新）、fsync()、Async（异步）

### sysfs / sys 文件系统
- **发音**：/sɪs-ef-es/
- **中文**：sys 文件系统
- **定义**：Linux 中的虚拟文件系统，将设备驱动模型信息暴露为文件
- **说明**：/sys 目录包含设备、驱动、总线、内核对象等信息；用于用户空间与内核交互
- **相关**：procfs、Virtual File System（虚拟文件系统）

### Timestamp / 时间戳
- **音标**：/ˈtɪmstæmp/
- **中文**：时间戳
- **定义**：记录文件时间信息的元数据
- **说明**：通常包括：访问时间（Access Time/atime）、修改时间（Modification Time/mtime）、状态改变时间（Change Time/ctime）
- **相关**：atime、mtime、ctime

### TLC (Triple-Level Cell) / 三层单元
- **中文**：三层单元
- **定义**：每个存储单元存储 3 位数据的闪存
- **说明**：TLC 成本低、容量大，但速度和寿命较低；现代消费级 SSD 主要使用 TLC
- **相关**：SLC、MLC、QLC、NAND

### tmpfs / tmp 文件系统
- **发音**：/tɛmp-ef-es/
- **中文**：tmp 文件系统
- **定义**：基于内存的临时文件系统
- **说明**：tmpfs 将文件存储在内存（和交换空间）中，速度快但断电丢失；用于 /tmp、/run 等
- **相关**：ramfs、Virtual File System（虚拟文件系统）

### Track / 磁道
- **音标**：/træk/
- **中文**：磁道
- **定义**：磁盘盘片上的同心圆轨道
- **说明**：数据存储在磁道上；磁道划分为多个扇区
- **相关**：Sector（扇区）、Cylinder（柱面）

### Transfer Time / 传输时间
- **中文**：传输时间
- **定义**：实际读写数据所需的时间
- **说明**：传输时间取决于数据量和磁盘传输速率；SSD 没有寻道时间和旋转延迟
- **相关**：Seek Time（寻道时间）、Rotational Latency（旋转延迟）

### TRIM
- **发音**：/trɪm/
- **中文**：修剪（命令）
- **定义**：允许操作系统通知 SSD 哪些数据块不再使用，可以被擦除的命令
- **说明**：TRIM 提高 SSD 的写入性能和寿命；没有 TRIM，SSD 不知道哪些块可以回收，写入性能会下降
- **相关**：SSD、Wear Leveling（磨损均衡）、Garbage Collection（垃圾回收）

### Truncate / 截断
- **音标**：/ˈtrʌŋkeɪt/
- **中文**：截断
- **定义**：将文件大小调整为指定值，超出部分丢弃或扩展为零填充
- **说明**：truncate() 系统调用截断文件；可以用于清空文件或预分配空间
- **相关**：File Size（文件大小）、ftruncate()

### UFS (Unix File System) / Unix 文件系统
- **中文**：Unix 文件系统
- **定义**：BSD Unix 系统的传统文件系统
- **说明**：UFS 也称为 FFS（Fast File System），引入Cylinder Group（柱面组）优化磁盘布局
- **相关**：BSD、FFS、ext2

### umask
- **发音**：/you-mask/
- **中文**：权限掩码
- **定义**：设置新创建文件和目录默认权限的掩码
- **说明**：文件默认权限 = 0666 & ~umask；目录默认权限 = 0777 & ~umask；如 umask 022 对应文件 644、目录 755
- **相关**：Permission（权限）、Mode（模式）

### Unmount / 卸载
- **音标**：/ˌʌnˈmaʊnt/
- **中文**：卸载
- **定义**：将文件系统从目录树中分离的操作
- **说明**：umount() 系统调用卸载文件系统；卸载前需要确保没有进程使用该文件系统
- **相关**：Mount（挂载）、Mount Point（挂载点）

### VFAT (Virtual File Allocation Table) / 虚拟文件分配表
- **中文**：虚拟文件分配表
- **定义**：FAT 文件系统的扩展，支持长文件名
- **说明**：VFAT 是 Windows 95 引入的，允许文件名长达 255 个字符；Linux 通过 VFAT 驱动访问 FAT 分区

## 文件操作

### Virtual File System (VFS) / 虚拟文件系统
- **中文**：虚拟文件系统
- **定义**：操作系统内核中的抽象层，为不同文件系统提供统一的接口
- **说明**：VFS 定义了通用文件系统接口（open、read、write 等），具体文件系统实现这些接口；Linux VFS 支持 ext4、XFS、NTFS 等
- **同义词**：Virtual Filesystem Switch
- **相关**：File System（文件系统）、Inode（索引节点）、Superblock（超级块）

### vnode
- **发音**：/viː-nɒd/
- **中文**：虚拟索引节点
- **定义**：VFS 中对 inode 的抽象，用于统一不同文件系统的文件表示
- **说明**：vnode 包含文件类型、操作函数指针、指向具体文件系统数据的指针等；每个打开的文件对应一个 vnode
- **相关**：VFS（虚拟文件系统）、Inode（索引节点）

### Volume / 卷
- **音标**：/ˈvɒljuːm/
- **中文**：卷
- **定义**：文件系统所在的逻辑存储单元
- **说明**：一个物理磁盘可以分为多个分区，每个分区是一个卷；逻辑卷管理（LVM）可以将多个物理卷组合成逻辑卷
- **相关**：Partition（分区）、Logical Volume（逻辑卷）

### Volume Group (VG) / 卷组
- **中文**：卷组
- **定义**：LVM 中由一个或多个物理卷组成的存储池
- **说明**：卷组中的空间可以分配给逻辑卷；可以动态添加物理卷扩展卷组
- **相关**：LVM、Physical Volume（物理卷）、Logical Volume（逻辑卷）

### Wear Leveling / 磨损均衡
- **中文**：磨损均衡
- **定义**：将写入操作均匀分布到所有闪存块的技术
- **说明**：闪存块有擦写次数限制，磨损均衡延长 SSD 寿命；分为动态磨损均衡和静态磨损均衡
- **相关**：SSD、Flash Memory（闪存）、TRIM

### Working Directory / 工作目录
- **中文**：工作目录（当前目录）
- **定义**：进程当前所在的目录
- **说明**：相对路径相对于工作目录解析；通过 cd 命令改变工作目录
- **同义词**：Current Directory
- **相关**：Relative Path（相对路径）、cd（改变目录）

### Write / 写入
- **音标**：/raɪt/
- **中文**：写入
- **定义**：将内存中的数据写入文件
- **说明**：write() 系统调用将数据写入当前文件偏移量处；可能触发延迟分配和延迟写入
- **相关**：Read（读取）、File Offset（文件偏移量）、Sync（同步）

### Write (w) / 写
- **中文**：写
- **定义**：修改文件内容或在目录中创建/删除文件的权限
- **相关**：Permission（权限）、Read（读）、Execute（执行）

### XFS
- **发音**：/eks-ef-es/
- **中文**：XFS 文件系统
- **定义**：SGI 开发的高性能日志文件系统，现为 Linux 的默认文件系统之一
- **说明**：XFS 适合大文件和大容量存储；支持在线扩容、延迟分配、并行 I/O；但不支持缩容
- **相关**：ext4、Btrfs、Journaling（日志记录）

### ZFS (Zettabyte File System) / ZFS 文件系统
- **发音**：/ziː-ef-es/
- **中文**：ZFS 文件系统
- **定义**：Sun Microsystems 开发的高级文件系统和逻辑卷管理器
- **说明**：ZFS 支持大容量存储、数据完整性校验、快照、克隆、压缩、去重、RAID-Z 等；Solaris/FreeBSD/Linux 支持
- **相关**：Btrfs、Volume Manager（卷管理器）、RAID
