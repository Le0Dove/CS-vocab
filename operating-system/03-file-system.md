# 03. 文件系统 (File System)

### Absolute Path / 绝对路径
- **中文**：绝对路径
- **定义**：从根目录开始的完整路径
- **说明**：Unix 中绝对路径以 / 开头，如 /home/user/file.txt；Windows 中以盘符开头，如 C:\Users\user\file.txt
- **相关**：Relative Path（相对路径）、Root Directory（根目录）

### Access Control / 访问控制
- **中文**：访问控制
- **定义**：限制用户对系统资源访问的机制
- **说明**：访问控制包括自主访问控制（DAC）、强制访问控制（MAC）、基于角色的访问控制（RBAC）
- **相关**：Permission（权限）、ACL（访问控制列表）、Security（安全）

### Access Control List (ACL) / 访问控制列表
- **中文**：访问控制列表
- **定义**：为文件/目录指定额外访问权限的列表
- **说明**：ACL 扩展了传统 Unix 权限模型，允许为多个用户和组设置细粒度权限
- **相关**：Permission（权限）、Access Control（访问控制）

### Allocation Size / 分配大小
- **中文**：分配大小
- **定义**：文件系统在磁盘上为文件分配的实际空间大小
- **说明**：分配大小是块大小的整数倍；可能大于文件实际大小（内部碎片）
- **相关**：File Size（文件大小）、Block Size（块大小）

### APFS (Apple File System) / 苹果文件系统
- **中文**：苹果文件系统
- **定义**：Apple 开发的用于 macOS、iOS 等系统的文件系统
- **说明**：APFS 针对闪存优化，支持快照、克隆、空间共享、加密、写时复制等；替代 HFS+
- **相关**：HFS+、Snapshot（快照）、Copy-on-Write（写时复制）

### Bitmap / 位图
- **音标**：/ˈbɪtmæp/
- **中文**：位图
- **定义**：用每一位表示一个块或 inode 是否被占用的数据结构
- **说明**：块位图（Block Bitmap）跟踪数据块的使用情况；inode 位图（Inode Bitmap）跟踪 inode 的使用情况
- **相关**：Free List（空闲列表）、Block（块）、Inode（索引节点）

### Block / 块
- **音标**：/blɒk/
- **中文**：块
- **定义**：文件系统中数据存储的基本单位
- **说明**：块大小通常为 1KB、2KB、4KB；文件按块分配存储空间；块是文件系统和磁盘 I/O 的基本单位
- **相关**：Block Size（块大小）、Sector（扇区）、Cluster（簇）

### Block Size / 块大小
- **中文**：块大小
- **定义**：文件系统中单个块的字节数
- **说明**：块大小影响文件系统的性能和空间利用率；大块减少碎片但增加内部碎片；小块节省空间但增加管理开销
- **相关**：Block（块）、Sector（扇区）

### Btrfs (B-tree File System) / B 树文件系统
- **发音**：/ˈbʌtər-fɪs/ 或 /ˈbiː-triː-fɪs/
- **中文**：B 树文件系统
- **定义**：Linux 的下一代文件系统，支持高级特性
- **说明**：Btrfs 支持写时复制（COW）、快照（Snapshot）、子卷（Subvolume）、校验和（Checksum）、RAID、压缩等
- **相关**：ZFS、ext4、Snapshot（快照）、Copy-on-Write（写时复制）

### Checksum / 校验和
- **音标**：/ˈtʃek-sʌm/
- **中文**：校验和
- **定义**：用于检测数据错误的计算值
- **说明**：文件系统使用校验和验证数据完整性；常见算法包括 CRC32、MD5、SHA-256
- **相关**：Data Integrity（数据完整性）、Error Detection（错误检测）

### chmod
- **发音**：/siː-mɒd/ 或 /tʃeɪ-mɒd/
- **中文**：改变权限（命令）
- **定义**：修改文件或目录权限的命令
- **说明**：chmod 可以使用符号模式（如 u+x, go-w）或八进制模式（如 755）；chmod() 是对应的系统调用
- **相关**：Permission（权限）、Mode（模式）

### Close / 关闭
- **音标**：/kloʊz/
- **中文**：关闭
- **定义**：断开进程与文件的连接，释放文件描述符
- **说明**：close() 系统调用关闭文件；关闭后文件描述符可以被重用；未关闭文件可能导致资源泄漏
- **相关**：Open（打开）、File Descriptor（文件描述符）

### Cluster / 簇
- **音标**：/ˈklʌstər/
- **中文**：簇
- **定义**：文件系统分配空间的最小单位，由连续的扇区组成
- **说明**：Windows FAT/NTFS 使用簇的概念；簇大小通常为 4KB、8KB、16KB
- **同义词**：Allocation Unit
- **相关**：Block（块）、Sector（扇区）

### Create / 创建
- **音标**：/kriˈeɪt/
- **中文**：创建
- **定义**：创建新文件的操作
- **说明**：creat() 系统调用创建新文件（或截断已有文件）并打开；open() 配合 O_CREAT 标志也可以创建文件
- **相关**：Open（打开）、Delete（删除）

### Dangling Link / 悬空链接
- **中文**：悬空链接
- **定义**：目标文件/目录已被删除的符号链接
- **说明**：访问悬空链接会报错；需要定期清理或更新符号链接
- **相关**：Symbolic Link（符号链接）

### Data Block / 数据块
- **中文**：数据块
- **定义**：存储文件实际数据的块
- **说明**：inode 中包含指向数据块的指针；大文件需要间接块（Indirect Block）指向更多数据块
- **相关**：Inode（索引节点）、Indirect Block（间接块）

### Data Integrity / 数据完整性
- **中文**：数据完整性
- **定义**：确保数据在存储和传输过程中不被损坏或篡改的属性
- **说明**：文件系统通过校验和、日志、写时复制等技术保证数据完整性
- **相关**：Checksum（校验和）、Journaling（日志记录）、Copy-on-Write（写时复制）

### Defragmentation / 碎片整理
- **音标**：/diːˌfræɡmenˈteɪʃn/
- **中文**：碎片整理
- **定义**：重新组织磁盘上的文件，使每个文件的数据连续存储
- **说明**：碎片整理减少文件访问的寻道时间；传统 HDD 需要碎片整理，SSD 不需要（且会缩短寿命）
- **同义词**：Defrag
- **相关**：Fragmentation（碎片化）、Disk（磁盘）

### Delete / 删除
- **音标**：/dɪˈliːt/
- **中文**：删除
- **定义**：删除文件或目录的操作
- **说明**：unlink() 系统调用删除文件（减少链接计数）；rmdir() 删除空目录；rm 命令删除文件/目录
- **相关**：Create（创建）、Remove（移除）、unlink()

### dentry
- **发音**：/diː-ɛntri/
- **中文**：目录项缓存
- **定义**：VFS 中缓存目录项（文件名到 inode 映射）的数据结构
- **说明**：dentry 缓存加速路径解析；目录项缓存（dentry cache/dcache）是 Linux 内核的重要缓存
- **相关**：VFS（虚拟文件系统）、Inode（索引节点）、Path（路径）

### Directory / 目录
- **音标**：/dəˈrektəri/
- **中文**：目录（文件夹）
- **定义**：用于组织和管理文件的特殊文件，包含文件名到文件位置的映射
- **说明**：目录可以嵌套形成树形结构；根目录（Root Directory）是文件系统的起点
- **同义词**：Folder（文件夹）
- **相关**：File（文件）、Path（路径）、Root Directory（根目录）

### Execute (x) / 执行
- **中文**：执行
- **定义**：执行文件（程序/脚本）或进入目录的权限
- **相关**：Permission（权限）、Read（读）、Write（写）

### ext4 (Fourth Extended File System) / 第四扩展文件系统
- **中文**：第四扩展文件系统
- **定义**：Linux 的默认文件系统，ext3 的后续版本
- **说明**：ext4 支持更大的文件（16TB）和文件系统（1EB）、Extents（连续块分配）、延迟分配、多块分配器、快速文件系统检查等
- **相关**：ext3、Extents、Linux

### Extent / 区段
- **音标**：/ɪkˈstent/
- **中文**：区段
- **定义**：文件系统中连续的块范围，作为一个分配单元
- **说明**：Extents 减少大文件的碎片化和管理开销；ext4、XFS 使用 extents 替代传统块指针
- **相关**：Block（块）、Contiguous Allocation（连续分配）

### FAT32 (File Allocation Table 32) / 32 位文件分配表
- **中文**：32 位文件分配表
- **定义**：FAT 文件系统的 32 位版本
- **说明**：FAT32 兼容性好，被几乎所有操作系统支持；但单文件最大 4GB，分区最大 2TB，无日志功能
- **相关**：FAT16、exFAT、NTFS

### File / 文件
- **音标**：/faɪl/
- **中文**：文件
- **定义**：存储在存储设备上的命名数据集合
- **说明**：文件是操作系统管理数据的基本单位；包含数据和元数据（文件名、大小、创建时间、权限等）
- **相关**：Directory（目录）、File System（文件系统）、Inode（索引节点）

### File Allocation Table (FAT) / 文件分配表
- **中文**：文件分配表
- **定义**：FAT 文件系统中用于记录文件块链接关系的表
- **说明**：FAT 表中每个条目对应一个块，存储下一个块的编号或结束标记；文件通过 FAT 表链式存储
- **相关**：FAT12、FAT16、FAT32、Cluster（簇）

### File Attribute / 文件属性
- **中文**：文件属性
- **定义**：文件的元数据信息
- **说明**：包括文件类型、大小、时间戳、权限、所有者等
- **相关**：Metadata（元数据）、Inode（索引节点）

### File Descriptor / 文件描述符
- **中文**：文件描述符
- **定义**：操作系统分配给已打开文件的非负整数标识符
- **说明**：标准输入（stdin）为 0，标准输出（stdout）为 1，标准错误（stderr）为 2；文件描述符是进程级资源
- **相关**：Open（打开）、Close（关闭）、File Description（文件描述）

### File Name / 文件名
- **中文**：文件名
- **定义**：标识文件的字符串
- **说明**：文件名在同一目录下必须唯一；不同文件系统对文件名长度和字符有不同限制
- **相关**：Extension（扩展名）、Path（路径）

### File Offset / 文件偏移量
- **中文**：文件偏移量
- **定义**：文件当前读写位置相对于文件开头的字节数
- **说明**：每次读写后偏移量自动前进；可以通过 seek 改变偏移量
- **相关**：Seek（定位）、Read（读取）、Write（写入）

### File Size / 文件大小
- **中文**：文件大小
- **定义**：文件包含的数据字节数
- **说明**：文件大小可能小于分配大小（Allocation Size），因为文件系统按块分配空间
- **相关**：Allocation Size（分配大小）、Block Size（块大小）

### File System / 文件系统
- **中文**：文件系统
- **定义**：操作系统中负责管理和组织文件存储、检索、命名的子系统
- **说明**：文件系统提供文件的逻辑视图，屏蔽物理存储细节；包括文件组织、存储空间管理、目录管理、文件保护等功能
- **相关**：File（文件）、Directory（目录）、Volume（卷）、Partition（分区）

### File Type / 文件类型
- **中文**：文件类型
- **定义**：文件的分类，如普通文件、目录、设备文件等
- **说明**：Unix 中文件类型包括：普通文件（-）、目录（d）、字符设备（c）、块设备（b）、符号链接（l）、管道（p）、套接字（s）
- **相关**：File（文件）、Extension（扩展名）、Magic Number（魔数）

### Fragmentation / 碎片化
- **中文**：碎片化
- **定义**：文件数据分散存储在磁盘不连续位置的状态
- **说明**：碎片化降低文件访问性能；文件系统通过预分配、延迟分配等技术减少碎片化
- **相关**：Defragmentation（碎片整理）、Allocation（分配）

### Group / 组
- **音标**：/ɡruːp/
- **中文**：组
- **定义**：用户的集合，用于批量管理权限
- **说明**：文件的组权限适用于该组的所有成员；chgrp 命令改变文件所属组
- **相关**：Owner（所有者）、Permission（权限）

### Hard Link / 硬链接
- **中文**：硬链接
- **定义**：指向文件 inode 的目录项，与原始文件共享相同的物理数据
- **说明**：硬链接与原始文件完全等价，删除其中一个不影响另一个；不能跨文件系统，不能链接目录
- **相关**：Symbolic Link（符号链接）、Inode（索引节点）、Link Count（链接计数）

### Indirect Block / 间接块
- **中文**：间接块
- **定义**：不存储文件数据，而是存储指向其他数据块指针的块
- **说明**：分为一级间接块（Single Indirect）、二级间接块（Double Indirect）、三级间接块（Triple Indirect）；用于支持大文件
- **相关**：Inode（索引节点）、Data Block（数据块）

### Indexed Allocation / 索引分配
- **中文**：索引分配
- **定义**：每个文件有一个索引块，存储该文件所有数据块的指针
- **说明**：索引分配支持直接访问，无外部碎片；Unix inode 使用索引分配（直接块+间接块）
- **相关**：Contiguous Allocation（连续分配）、Linked Allocation（链接分配）、Inode（索引节点）

### Inode / 索引节点
- **音标**：/ˈaɪnoʊd/
- **中文**：索引节点
- **定义**：Unix 文件系统中存储文件元数据的数据结构
- **说明**：每个文件对应一个 inode；inode 包含文件类型、权限、所有者、时间戳、大小、数据块指针等；文件名存储在目录中
- **相关**：File（文件）、Directory（目录）、Data Block（数据块）

### Journal / 日志
- **音标**：/ˈdʒɜːrnl/
- **中文**：日志
- **定义**：记录文件系统操作的日志，用于故障恢复
- **说明**：日志文件系统（Journaling File System）在修改元数据前先写日志；系统崩溃后可以通过日志恢复一致性
- **相关**：Journaling（日志记录）、Transaction（事务）、Recovery（恢复）

### Journaling / 日志记录
- **中文**：日志记录
- **定义**：将文件系统操作记录到日志中的技术
- **说明**：日志记录模式包括：写日志（Writeback，只记录元数据）、有序（Ordered，数据先于元数据写入）、数据（Data，记录数据和元数据）
- **相关**：Journal（日志）、File System（文件系统）、ext3、ext4

### Linked Allocation / 链接分配
- **定义**：将文件数据存储在不连续的块中，每个块包含指向下一个块的指针
- **说明**：链接分配无外部碎片，但随机访问慢，指针占用空间；FAT 是链接分配的变体
- **相关**：Contiguous Allocation（连续分配）、Indexed Allocation（索引分配）、FAT

### Master File Table (MFT) / 主文件表
- **中文**：主文件表
- **定义**：NTFS 文件系统中存储所有文件和目录元数据的特殊文件
- **说明**：MFT 由多个记录组成，每个记录 1KB；小文件的数据直接存储在 MFT 记录中（常驻属性），大文件的数据存储在独立块中（非常驻属性）
- **相关**：NTFS、Inode（索引节点）、Attribute（属性）

### Metadata / 元数据
- **音标**：/ˈmetədeɪtə/
- **中文**：元数据
- **定义**：描述文件属性和管理信息的数据
- **说明**：元数据包括文件名、大小、创建时间、修改时间、访问时间、权限、所有者等
- **相关**：Inode（索引节点）、File Attribute（文件属性）

### Mirror / 镜像
- **音标**：/ˈmɪrər/
- **中文**：镜像
- **定义**：将数据完整复制到另一个磁盘的技术
- **说明**：镜像提供冗余，一个磁盘故障时数据不丢失；RAID 1 使用镜像
- **相关**：RAID、Redundancy（冗余）

### Mode / 模式
- **音标**：/moʊd/
- **中文**：模式
- **定义**：Unix 中文件权限和类型的组合
- **说明**：模式通常用八进制表示，如 755（rwxr-xr-x）、644（rw-r--r--）；包含文件类型和权限位
- **相关**：Permission（权限）、umask

### Mount / 挂载
- **音标**：/maʊnt/
- **中文**：挂载
- **定义**：将文件系统附加到目录树中的操作
- **说明**：mount() 系统调用将设备上的文件系统挂载到指定挂载点；卸载使用 umount()
- **相关**：Unmount（卸载）、Mount Point（挂载点）、File System（文件系统）

### Mount Point / 挂载点
- **中文**：挂载点
- **定义**：文件系统中用于挂载其他文件系统的目录
- **说明**：挂载后，访问挂载点目录就是访问被挂载的文件系统根目录；如 /mnt、/media
- **相关**：Mount（挂载）、Root Directory（根目录）

### NTFS (New Technology File System) / 新技术文件系统
- **中文**：新技术文件系统
- **定义**：Windows 的默认文件系统
- **说明**：NTFS 支持大文件、权限控制、加密、压缩、磁盘配额、硬链接、符号链接、日志记录等
- **相关**：FAT32、Windows、MFT（主文件表）

### Open / 打开
- **音标**：/ˈoʊpən/
- **中文**：打开
- **定义**：建立进程与文件之间的连接，返回文件描述符
- **说明**：open() 系统调用打开文件，操作系统检查权限、定位文件、分配文件描述符；打开后可以进行读写操作
- **相关**：Close（关闭）、File Descriptor（文件描述符）、open()（系统调用）

### Owner / 所有者
- **音标**：/ˈoʊnər/
- **中文**：所有者
- **定义**：文件的拥有用户
- **说明**：文件的所有者可以改变文件权限；chown 命令改变所有者
- **相关**：Group（组）、Permission（权限）

### Partition / 分区
- **音标**：/pɑːrˈtɪʃn/
- **中文**：分区
- **定义**：将物理磁盘划分为多个独立区域的操作
- **说明**：分区表记录分区信息；MBR（主引导记录）和 GPT（GUID 分区表）是两种分区方案
- **相关**：Volume（卷）、MBR、GPT

### Path / 路径
- **音标**：/pæθ/
- **中文**：路径
- **定义**：标识文件或目录在文件系统中位置的字符串
- **说明**：绝对路径（Absolute Path）从根目录开始；相对路径（Relative Path）从当前目录开始
- **相关**：Absolute Path（绝对路径）、Relative Path（相对路径）、Working Directory（工作目录）

### Permission / 权限
- **音标**：/pərˈmɪʃn/
- **中文**：权限
- **定义**：控制用户对文件/目录访问操作的规则
- **说明**：Unix 权限包括读（Read）、写（Write）、执行（Execute），分别对应所有者（Owner）、组（Group）、其他（Other）
- **相关**：Access Control（访问控制）、ACL（访问控制列表）、Mode（模式）

### RAID (Redundant Array of Independent Disks) / 独立磁盘冗余阵列
- **发音**：/reɪd/
- **中文**：独立磁盘冗余阵列
- **定义**：将多个物理磁盘组合成一个逻辑单元，以提高性能或可靠性的技术
- **说明**：RAID 级别包括：RAID 0（条带化，性能高无冗余）、RAID 1（镜像，冗余高）、RAID 5（分布式奇偶校验）、RAID 6（双奇偶校验）、RAID 10（镜像+条带化）
- **相关**：Disk（磁盘）、Redundancy（冗余）、Stripe（条带化）、Mirror（镜像）

### Read / 读取
- **音标**：/riːd/
- **中文**：读取
- **定义**：从文件中读取数据到内存
- **说明**：read() 系统调用从当前文件偏移量处读取指定字节数的数据；读取后偏移量前进
- **相关**：Write（写入）、File Offset（文件偏移量）

### Relative Path / 相对路径
- **中文**：相对路径
- **定义**：相对于当前工作目录的路径
- **说明**：不以 / 或盘符开头；使用 .（当前目录）和 ..（父目录）表示相对位置
- **相关**：Absolute Path（绝对路径）、Working Directory（工作目录）

### Root Directory / 根目录
- **中文**：根目录
- **定义**：文件系统的最顶层目录
- **说明**：Unix 中根目录为 /；Windows 中每个分区有自己的根目录（如 C:\）
- **相关**：Path（路径）、Directory（目录）

### Seek / 定位
- **音标**：/siːk/
- **中文**：定位
- **定义**：改变文件的当前读写位置（偏移量）
- **说明**：lseek() 系统调用可以绝对定位（SEEK_SET）、相对定位（SEEK_CUR）、尾部定位（SEEK_END）
- **相关**：File Offset（文件偏移量）、Read（读取）、Write（写入）

### Setgid / 设置组 ID
- **发音**：/set-gee-eye-dee/
- **中文**：设置组 ID
- **定义**：文件权限位，允许用户以文件所属组的权限执行该文件，或在目录中创建的文件继承目录的组
- **相关**：Setuid（设置用户 ID）、Permission（权限）

### Setuid / 设置用户 ID
- **发音**：/set-you-eye-dee/
- **中文**：设置用户 ID
- **定义**：文件权限位，允许用户以文件所有者的权限执行该文件
- **说明**：Setuid 位用 s 表示（如 rwsr-xr-x）；常用于 passwd、ping 等需要特权的程序；安全风险较大
- **相关**：Setgid（设置组 ID）、Sticky Bit（粘滞位）、Permission（权限）

### Snapshot / 快照
- **音标**：/ˈsnæpʃɒt/
- **中文**：快照
- **定义**：文件系统或卷在某一时刻的只读副本
- **说明**：快照使用写时复制（COW）技术，初始不占用额外空间；常用于备份和数据保护
- **相关**：Copy-on-Write（写时复制）、Backup（备份）、LVM、Btrfs、ZFS

### SSD (Solid State Drive) / 固态驱动器
- **中文**：固态驱动器
- **定义**：使用闪存（Flash Memory）存储数据的电子设备
- **说明**：SSD 速度快、无噪音、抗震，但价格较高、有写入次数限制；现代系统首选存储设备
- **相关**：HDD、Flash Memory（闪存）、NAND

### Sticky Bit / 粘滞位
- **中文**：粘滞位
- **定义**：目录权限位，限制只有文件所有者才能删除该目录中的文件
- **说明**：粘滞位用 t 表示（如 rwxrwxrwt）；常用于 /tmp 等公共目录，防止用户删除他人的临时文件
- **相关**：Permission（权限）、/tmp

### Stripe / 条带化
- **音标**：/straɪp/
- **中文**：条带化
- **定义**：将数据分块分散存储到多个磁盘的技术
- **说明**：条带化提高读写性能；RAID 0 使用纯条带化，RAID 5/6 结合条带化和奇偶校验
- **相关**：RAID、Mirror（镜像）

### Superblock / 超级块
- **音标**：/ˈsuːpərblɒk/
- **中文**：超级块
- **定义**：文件系统中存储文件系统整体元数据的结构
- **说明**：超级块包含块大小、块总数、空闲块数、inode 总数、空闲 inode 数、挂载时间等；是文件系统的关键结构
- **相关**：File System（文件系统）、Block（块）、Inode（索引节点）

### Symbolic Link / 符号链接
- **中文**：符号链接（软链接）
- **定义**：包含目标文件/目录路径的特殊文件
- **说明**：符号链接可以跨文件系统，可以链接目录；删除目标文件后符号链接成为悬空链接（Dangling Link）
- **同义词**：Soft Link、Symlink
- **相关**：Hard Link（硬链接）、Dangling Link（悬空链接）

### Timestamp / 时间戳
- **音标**：/ˈtɪmstæmp/
- **中文**：时间戳
- **定义**：记录文件时间信息的元数据
- **说明**：通常包括：访问时间（Access Time/atime）、修改时间（Modification Time/mtime）、状态改变时间（Change Time/ctime）
- **相关**：atime、mtime、ctime

### umask
- **发音**：/you-mask/
- **中文**：权限掩码
- **定义**：设置新创建文件和目录默认权限的掩码
- **说明**：文件默认权限 = 0666 & ~umask；目录默认权限 = 0777 & ~umask；如 umask 022 对应文件 644、目录 755
- **相关**：Permission（权限）、Mode（模式）

### Unmount / 卸载
- **音标**：/ˌʌnˈmaʊnt/
- **中文**：卸载
- **定义**：将文件系统从目录树中分离的操作
- **说明**：umount() 系统调用卸载文件系统；卸载前需要确保没有进程使用该文件系统
- **相关**：Mount（挂载）、Mount Point（挂载点）

### Virtual File System (VFS) / 虚拟文件系统
- **中文**：虚拟文件系统
- **定义**：操作系统内核中的抽象层，为不同文件系统提供统一的接口
- **说明**：VFS 定义了通用文件系统接口（open、read、write 等），具体文件系统实现这些接口；Linux VFS 支持 ext4、XFS、NTFS 等
- **同义词**：Virtual Filesystem Switch
- **相关**：File System（文件系统）、Inode（索引节点）、Superblock（超级块）

### vnode
- **发音**：/viː-nɒd/
- **中文**：虚拟索引节点
- **定义**：VFS 中对 inode 的抽象，用于统一不同文件系统的文件表示
- **说明**：vnode 包含文件类型、操作函数指针、指向具体文件系统数据的指针等；每个打开的文件对应一个 vnode
- **相关**：VFS（虚拟文件系统）、Inode（索引节点）

### Volume / 卷
- **音标**：/ˈvɒljuːm/
- **中文**：卷
- **定义**：文件系统所在的逻辑存储单元
- **说明**：一个物理磁盘可以分为多个分区，每个分区是一个卷；逻辑卷管理（LVM）可以将多个物理卷组合成逻辑卷
- **相关**：Partition（分区）、Logical Volume（逻辑卷）

### Working Directory / 工作目录
- **中文**：工作目录（当前目录）
- **定义**：进程当前所在的目录
- **说明**：相对路径相对于工作目录解析；通过 cd 命令改变工作目录
- **同义词**：Current Directory
- **相关**：Relative Path（相对路径）、cd（改变目录）

### Write / 写入
- **音标**：/raɪt/
- **中文**：写入
- **定义**：将内存中的数据写入文件
- **说明**：write() 系统调用将数据写入当前文件偏移量处；可能触发延迟分配和延迟写入
- **相关**：Read（读取）、File Offset（文件偏移量）、Sync（同步）

### XFS
- **发音**：/eks-ef-es/
- **中文**：XFS 文件系统
- **定义**：SGI 开发的高性能日志文件系统，现为 Linux 的默认文件系统之一
- **说明**：XFS 适合大文件和大容量存储；支持在线扩容、延迟分配、并行 I/O；但不支持缩容
- **相关**：ext4、Btrfs、Journaling（日志记录）

### ZFS (Zettabyte File System) / ZFS 文件系统
- **发音**：/ziː-ef-es/
- **中文**：ZFS 文件系统
- **定义**：Sun Microsystems 开发的高级文件系统和逻辑卷管理器
- **说明**：ZFS 支持大容量存储、数据完整性校验、快照、克隆、压缩、去重、RAID-Z 等；Solaris/FreeBSD/Linux 支持
- **相关**：Btrfs、Volume Manager（卷管理器）、RAID
